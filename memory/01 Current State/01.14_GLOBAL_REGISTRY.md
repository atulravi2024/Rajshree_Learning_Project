## 🏗️ CORE STATE VARIABLES
These variables track the user's progress through the 5-step navigation system.

| Variable | Type | Initial Value | Managed By | Purpose |
|:---|:---:|:---:|:---|:---|
| `window.selectedMain` | `String` | `null` | `router.js` | Top-level category (e.g., 'varnamala') |
| `window.selectedTitle` | `String` | `null` | `router.js` | Sub-category (e.g., 'swar') |
| `window.selectedCategory` | `String`| `null` | `router.js` | Final learning category key |
| `window.currentIndex` | `Number` | `0` | `slideshow/*.js` | Current vertical index in Learning Zone |
| `window.currentLayout` | `String` | `'1'` | `settings-modal.js`| Active display mode ('1', '3', 'all', 'fit') |

---

## 🛠️ USAGE RULES
1.  **Read-Only in UI**: UI components (`display.js`) should rarely modify these directly; use `router.js` or `settings-modal.js` functions.
2.  **Initialization**: All variables MUST be initialized in `init.js` or at the top of their respective core modules.
3.  **Conflict Prevention**: Always check this registry before adding a new `window.` property.
