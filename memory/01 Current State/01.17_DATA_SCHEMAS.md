## 🏗️ CORE DATA SCHEMAS
Every data module in `js/data/` must adhere to these structural standards to ensure compatibility with `display.js`.

### 1. 📖 Alphabet Schema (Swar, Vyanjan, etc.)
Used when learning individual characters.
```javascript
{
  letter: "अ",      // The character to display
  word: "अनार",     // The Hindi label
  icon: "🍎",      // The primary visual cue (Emoji)
  audio: "a_anar"   // Filename WITHOUT .mp3 extension
}
```

### 2. 🧮 Number Schema (Ginti)
Used for counting and tables.
```javascript
{
  number: "1",      // The numeral
  word: "एक",        // The Hindi label
  icon: "1️⃣",       // Primary icon (or dynamic array)
  audio: "num_1"    // Filename
}
```

### 3. 🌎 General Category (Mera Sansar, Objects, etc.)
Used for real-world objects and vocabulary.
```javascript
{
  word: "पिता",      // Hindi word
  english: "Father", // Primary identifier (often same as filename)
  icon: "👨",       // Visual cue
  audio: "father"    // Filename
}
```

### 4. 🔴 Special Rendering Keys
- `type: 'red-circle'`: Triggers the double-circle "Khali" icon in `display.js`.
- `textOnly: true`: Disables icon rendering for purely linguistic cards.

---

## 🛠️ VALIDATION CHECKLIST
- [ ] **No Extensions**: `audio` keys MUST NOT include `.mp3`.
- [ ] **String Values**: All numeric values in `number` or `letter` should be typed as `Strings`.
- [ ] **Icon Mandatory**: Every object MUST have an `icon` or a `type` override.
