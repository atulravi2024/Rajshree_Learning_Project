## 🏗️ ARCHITECTURE DECISIONS

### Why Memory Was Split (v14.0)
- Single 26KB file was too large for efficient processing.
- Split into a "Frequency × Layer" hybrid (Core rules + On-demand detail files).
- Ensures minimal token usage per session while maintaining 100% detail coverage.

### Why CSS Was Modularized (v13.0)
- Monolithic `style.css` (19KB) was replaced with 10 specialized files.
- Prevents regressions in core layouts when adding new theme-specific features.
- Enables conditional loading for performance optimization.
