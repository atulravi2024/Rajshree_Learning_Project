# 🗄️ MOBILE STATE PERSISTENCE (v1.0)

## 1. Registry: `localStorage`
The mobile app maintains a synchronized state across sessions using 50+ unique keys.

### A. Session & Core State
| Key | Purpose | Default |
|:---|:---|:---|
| `mobile_session_active` | App initialized & Welcome bypassed | `false` |
| `mobile_currentCategory`| Active registry key (e.g., `swar`) | `swar` |
| `mobile_currentIndex` | Active card offset | `0` |

### B. Audio & Volume Settings
- `mobile_vol_master`, `mobile_vol_music`, `mobile_vol_sfx`, `mobile_vol_content`.
- `mobile_voice_profile`: (`neerja`, `swara`, `madhur`).
- `mobile_playback_speed`: (0.25x - 2.0x).
- `mobile_bg_music`, `mobile_sfx`: (Booleans).

### C. Visuals & Layout
- `mobile_theme_primary`: (`pink`, `blue`, `orange`, etc.).
- `mobile_dark_mode`, `mobile_auto_dark`, `mobile_reduce_motion`.
- `mobile_view_mode`: (`flashcard`, `grid`, `three`).
- `mobile_flashcard_nav_dir`: (`horizontal`, `vertical`).
- `mobile_grid_nav_dir`: (`horizontal`, `vertical`).
- `mobile_three_nav_dir`: (`horizontal`, `vertical`).
- `mobile_show_bottom_nav`, `mobile_autohide_nav`.

### D. Security & Control
- `mobile_child_lock`: Global interceptor toggle.
- `mobile_parent_pin`, `mobile_admin_pin`, `mobile_dev_pin`, `mobile_designer_pin`.
- `mobile_pin_visible`: Whether PIN digits are masked or shown in the modal.

---

## 2. Transient Logic
- **`sessionStorage.activeSettingsTab`**: Persists the last active tab in the Settings panel during the current browser session.
