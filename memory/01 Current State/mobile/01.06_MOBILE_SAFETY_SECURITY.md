# 🛡️ MOBILE SAFETY & SECURITY (v1.0)

## 1. Child Lock System
The mobile application features a robust "Child Lock" (`mobile_child_lock`) that restricts access to navigation and settings.

### A. Global Interceptor
- **Logic**: If `mobile_child_lock` is `true`, a global listener in `settings_core.js` intercepts all `pointerdown`, `change`, and `input` events.
- **Exceptions**: Allows tab switches if they are guarded by PIN logic.

## 2. Parent & Admin Protection
### A. PIN Pad (`#pin-modal`)
- **Capacity**: 4 digits.
- **Verification**: Cross-references `localStorage` keys (e.g., `mobile_parent_pin`).
- **Confirmation Flow**: Setup mode requires entering the new PIN twice for verification.

### B. Math Pad (`#math-modal`)
- **Challenge**: Generates random simple arithmetic (Addition, Subtraction, Multiplication).
- **Gatekeeper**: Used as an alternative to PINs to verify adult supervision.

## 3. Granular Safety Locks
Located in the Admin Tab, these locks control specific mobile behaviors:
- **Navigation Locks**: `mobile_lock_nav`, `mobile_lock_home`.
- **System Locks**: `mobile_lock_vol`, `mobile_lock_context` (Right Click), `mobile_lock_back`.
- **Audio Contextual Groups**: Locks navigation and autoplay only while audio is actively playing.
