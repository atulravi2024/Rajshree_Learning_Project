## 📐 CORE ARCHITECTURE

### 1. Responsive & Routing
- **Auto-Redirect**: If `window.innerWidth > 768`, the app redirects to the desktop version.
- **Session Persistence**: Uses `localStorage` (key: `mobile_session_active`) to restore the learning state (Flashcard vs Welcome).

### 2. UI Components & Interaction
- **Overlay System**: Multi-step menu (`menu-engine.js`) and persistent Settings (`settings_core.js`).
- **Navigation Navbar**: Fixed bottom bar with boundary-aware logic (Prev/Next buttons disable at deck ends).
- **Gestures**: Built-in swipe detection (Horizontal/Vertical based on Layout mode).

### 3. Layout Modes (3-in-1 Logic)
| Mode | Description | Nav Direction | Scaling |
|:---|:---|:---|:---|
| **Flashcard** | Single card focus | Dynamic (H/V) | Full Scale |
| **Grid** | Category overview | Dynamic (H/V) | Thumbnails |
| **Three-in-One**| 3 cards at once | Dynamic (H/V) | 0.4x Scale |

### 4. Engine Infrastructure
- **ThemeEngine**: Handles dynamic CSS variable injection and Dark Mode.
- **MenuEngine**: Orchestrates category discovery steps with auditory intros.
- **i18nEngine**: Manages real-time UI translation (Hindi, English, Hinglish).
- **SettingsEngine**: Manages 50+ configuration keys with PIN/Math security.
