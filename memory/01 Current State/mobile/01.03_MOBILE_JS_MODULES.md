# ⚙️ MOBILE JS MODULES (v1.0)

## 1. Engine Core: `js/`
The mobile application uses a specialized logic system designed for performance on low-power devices.

### A. Initialization Flow (`main.js`)
1. **State Boot**: Loads configuration from `localStorage` into `window.state`.
2. **Data Sync**: Populates `state.data` from the `window.RAJSHREE_DATA` container.
3. **UI Localization**: Calls `i18n.js` to translate the DOM based on active settings.
4. **Layout Setup**: Configures Navbar visibility and container directions.
5. **Session Restore**: Checks `mobile_session_active` to decide whether to show the Welcome screen or jump to the last card.

### B. Navigation Engine
- **`next()` / `prev()`**: Boundary-aware card switching.
- **`flipCard(card, audio)`**: Handles 3D animation and triggers audio playback.
- **`toggleAutoplay()`**: Manages the persistent autoplay state and triggers auto-advance.

### C. The Multi-Step Menu (`menu-engine.js`)
- Uses a history stack (`menuState.history`) to allow seamless "Back" navigation through category steps.
- **Step 1**: Root Selection.
- **Step 2**: Sub-category Selection.
- **Step 3**: Final Registry Selection.

---

## 2. Shared Data Interface
Mobile logic is fully decoupled from the raw data. It expects:
- `item.letter`: The primary large text.
- `item.word`: The Hindi word (hi-IN).
- `item.emoji`: The card's visual anchor.
- `item.audio`: Relative path from `AUDIO_BASE_PATH`.

---

## 3. Global State Registry (`window.state`)
| Key | Type | Purpose |
|:---|:---|:---|
| `currentIndex` | Integer | Active card pointer. |
| `currentCategoryName` | String | Active deck registry key. |
| `isAutoplay` | Boolean | Continuous playback state. |
| `vMaster` | Float | Base multiplier for all audio. |
| `viewMode` | String | layout selector (flashcard, grid, three). |
