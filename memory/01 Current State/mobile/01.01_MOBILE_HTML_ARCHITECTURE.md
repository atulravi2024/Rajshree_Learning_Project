# 📑 MOBILE HTML ARCHITECTURE (v1.0)

## 1. Learning Hub: `index.html`
The core application shell designed for single-handed mobile use.

### A. Viewport & Redirection
- **Scale**: `width=device-width, initial-scale=1.0` (Native mobile feel).
- **Auto-Router**: 
  ```javascript
  if (window.innerWidth > 768) window.location.replace('../desktop/index.html');
  ```
  This ensures mobile users don't see the complex desktop grid on small screens.

### B. Structural Overlays
- `#menu-overlay`: Full-screen multi-step category selection. Use `data-menu-style="classic"` for styling hooks.
- `#welcome-screen`: A premium landing page with `.welcome-card` and the animated `.start-orb`.
- `#master-mobile`: The primary learning container. Uses `.hidden` for state management.

### C. Persistent Navigation (`.mobile-nav`)
- **Left**: `#mobile-prev` (Navigate backward).
- **Center**: `#mobile-home` (Reset state and Go Home).
- **Right**: `#mobile-next` (Navigate forward).
- **Auxiliary**: `#mobile-autoplay-nav` (Visual toggle) and `#mobile-settings` (Direct link to `settings.html`).

## 2. Configuration Hub: `settings.html`
A high-depth administrative interface for parents and experts.

### A. Tabbed Navigation (`#categoryTabs`)
- Uses `data-target` to switch between:
  - `cat-kids`: Core audio/visual settings.
  - `cat-parent`: Learning goals and screen time.
  - `cat-admin`: Global safety locks and security.
  - `cat-developer`: Debugging and performance.
  - `cat-designer`: Theme customization and vibrancy.
  - `cat-help`: System documentation.

### B. Accordion System (`.category-block`)
- Every setting group is wrapped in an accordion header (`.section-header`).
- Optimized for mobile scrolling with `.expanded` state management.

### C. Security Modals
- `#pin-modal`: 4-digit keypad for parent/admin verification.
- `#math-modal`: Equation solver (e.g., `24 + 15 = ?`) for child-gate protection.
