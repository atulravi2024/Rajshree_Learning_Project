# 🧭 MOBILE NAVIGATION & MENU (v1.0)

## 1. Multi-Step Menu Engine (`menu-engine.js`)
The menu is designed for hierarchical discovery without overwhelming the screen.

### A. Navigation Hierarchy
- **Step 1 (Root)**: High-level categories (Varnamala, Ganit, Mera Sansar, Khel-Kood).
- **Step 2 (Branch)**: Sub-categories (e.g., Swar/Vyanjan, Ginti/Pahade).
- **Step 3 (Leaf)**: Specific deck registry entry (e.g., Swar, Numbers 1-100).

### B. Navigation History
- A history stack (`menuState.history`) tracks every step.
- Clicking "Back" pops the stack and reveals the previous container.

## 2. Floating Learning Navbar (`.mobile-nav`)
- **Boundary Logic**: The `updateNavigationUI()` function disables navigation buttons if the `currentIndex` is at the exact Start or End of the data array.
- **Visual Feedback**: The active navigation mode (e.g., Autoplay) is highlighted with an `.active-glow` effect.
- **Auto-Hide Logic**:
  - Hides after 5 seconds of inactivity.
  - Re-appears when touching the bottom 15% of the viewport.
