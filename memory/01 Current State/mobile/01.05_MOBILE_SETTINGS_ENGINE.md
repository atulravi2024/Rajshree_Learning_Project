# 🛠️ MOBILE SETTINGS ENGINE (v1.0)

## 1. Core Logic: `settings_core.js`
The settings engine handles 50+ configuration keys with a focus on UI responsiveness and persistence.

### A. Initialization Sequence
- **Load Logic**: `loadSettings()` reads all `localStorage` keys and synchronizes them with the DOM elements (inputs, switches, badges).
- **Control Sync**: Automatically updates value badges (e.g., `80%`) next to labels for slider inputs.

### B. Interactive UI Systems
- **Vertical Draggable**: Allows "flick" scrolling on mobile by hijacking `pointerdown` events on the container.
- **Horizontal Draggable Tabs**: The category tab bar (`.category-tabs`) supports momentum-style horizontal dragging.
- **Accordion Logic**: `initAccordion()` manages `.expanded` and `.focused` states for setting blocks.

## 2. Setting Segregation
| Tab ID | Content |
|:---|:---|
| `cat-kids` | Audio volumes, Speeds, Visual effects, Dark mode. |
| `cat-parent` | Screen timer, Break reminders, Difficulty, Daily goals. |
| `cat-admin` | Security masters, Safety locks, PIN management. |
| `cat-designer` | Theme colors, Glass quality, Vibrancy levels. |
| `cat-developer` | Grid navigation directions, Layout modes, Nav visibility. |

## 3. Persistent Sync
- All settings use the `mobile_` prefix in `localStorage` to avoid collisions with the desktop application.
- Example keys: `mobile_vol_master`, `mobile_autoplay_delay`, `mobile_theme_primary`.
