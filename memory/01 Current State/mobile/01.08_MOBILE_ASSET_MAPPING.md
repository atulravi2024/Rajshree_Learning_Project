# 🖼️ MOBILE ASSET MAPPING (v1.0)

## 1. Directory Context
Mobile shares the master `assets/` repository but utilizes specific paths for system-level feedback.

### A. UI Sounds & Feedback
- **Interaction SFX**: `assets/audio/system/effects/storm-wind.mp3` (Played on all major navigation/button events).
- **Celebration Reward**: `assets/audio/system/effects/reward_excellent.mp3` (Played upon category completion).
- **Profile Welcomes**: `assets/audio/system/welcome/welcome_[profile].mp3` (Triggered on app start).

### B. Navigation & Control Icons
Icons are largely handled via Emojis and SVGs within the HTML/JS for zero extraction overhead.
- **Home**: 🏠
- **Settings**: ⚙️
- **Autoplay**: 🔄
- **Category Icons**: 📖 (Varnamala), 🧮 (Ganit), 🌎 (Mera Sansar), 🎮 (Games).

## 2. Dynamic Asset Paths
Logic expects assets at `../assets/` relative to the `mobile/` directory.
- **Audio Base**: `window.AUDIO_BASE_PATH = '../assets/audio/';`
- **Image Support**: Card objects can contain an optional `image` key pointing to a relative path (e.g., `../assets/img/varnamala/swar/a.png`).
