## đźŹ—ď¸Ź OVERVIEW
Child safety is the project's highest priority. The system is designed to prevent children from accidentally changing settings, exiting the app, or spamming UI elements. All safety logic is centralized in `child-safety.js` and integrated with `helpers.js`.

---

## đź›‚ THE PARENTAL GATE (Time-Locked Actions)
To prevent accidental access to critical menus, specific buttons require a **3-second continuous hold** to activate.

| Feature | Trigger ID | Logic |
|:---|:---|:---|
| **Project Settings** | `#nav-settings` | 3s hold required. |
| **Stop Slideshow** | `#nav-slideshow` | 3s hold required (only when active). |

- **Visual Feedback**: The system sets a CSS variable `--hold-progress` (0 to 100%) on the element.
- **Auto-Bypass**: If `isSafetyEnabled` is `false`, the gate is bypassed (one-click access).

---

## đź› ď¸Ź INTERACTION CONTROLS (Anti-Spam)
The `ChildSafetyLock` object manages UI locking to ensure stability and focus.

### 1. Click Spam Guard
- **Threshold**: **400ms**.
- **Action**: Clicks occurring faster than this are ignored and logged as "Interaction blocked: Spam detection".

### 2. Navigation Throttle
- **Threshold**: **1000ms** (1 second).
- **Action**: In Kids Mode, manual card skipping (`next`/`prev`) is throttled to once per second to prevent rapid flashing.

### 3. Playback Locking
- **Lock**: UI is locked automatically via `ChildSafetyLock.lock()` when audio begins.
- **Unlock**: UI is released via `ChildSafetyLock.unlock()` when audio ends or fails.

## đź’» BROWSER LOCKDOWN (Safety Mode)
When Safety Mode is enabled, the browser is restricted to a "Kids Kiosk" environment.

### 1. Keyboard & Input Blocks
- **Right-Click**: Disabled (`contextmenu`).
- **Dev Tools**: Blocks `F12`, `Ctrl+Shift+I`, `Ctrl+Shift+J`, `Ctrl+Shift+C`.
- **System**: Blocks Page Source (`Ctrl+U`), Save (`Ctrl+S`), Print (`Ctrl+P`).
- **Input**: Disables text selection across the entire body.

### 2. Zoom Prevention
- **Scroll Zoom**: Blocks `Ctrl + Wheel`.
- **Keyboard Zoom**: Blocks `Ctrl + ( + / - / = / 0 )`.
- **Mobile Zoom**: Blocks pinch-to-zoom (multi-touch) and double-tap to zoom.

### 3. Auto Full-Screen
- Requests full-screen mode on the first user interaction.
- Automatically re-enters full-screen if exited (while safety is enabled).

## đź”‘ DEVELOPER ACCESS & BYPASS

### 1. The Master Bypass Code
- **Shortcut**: **`Ctrl + Alt + Shift + D`**
- **Action**: Toggles `isSafetyEnabled`.

### 2. URL Parameters
- Adding **`?dev=true`** to the URL disables safety features on load for debugging.

### 3. Mode-Based Volume Control
The system shifts volume automatically when switching modes to protect hearing:
- **Child Mode (ON)**: **100% Volume** (Normal for kids).
- **Dev Mode (OFF)**: **25% Volume** (To reduce noise during development).
