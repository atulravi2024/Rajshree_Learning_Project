## 📂 ADDING A NEW LEARNING CATEGORY
To add a new category (e.g., "Space") follow this exact sequence:

### 1. Data Module (`frontend/src/js/data/`)
- Create a new JS file (e.g., `space.js`).
- Use the standard array format: `const SPACE_DATA = [ { letter: '🌍', word: 'पृथ्वी', audio: 'space_earth.mp3' }, ... ];`
- Export it globally: `window.SPACE_DATA = SPACE_DATA;`

### 2. HTML Registration (`index.html`)
- Add the `<script>` tag in the **Data Modules** section (Line ~130).
- Ensure it loads *before* `core/init.js`.

### 3. Route Mapping (`frontend/src/js/navigation/router.js`)
- Update `showMainCategory()` or `showSubCategory()` to include the new key.
- Add the corresponding icon to `iconMap` in `finishSetup()`.
- Add intro audio calling logic: `playSound('system/intros/final_space.mp3')`.

### 4. Memory Sync (The most important step)
- **01.04 ASSETS**: Update the file count for the relevant folder.
- **01.08 ICONS**: Map the new icons used in step 3/4.
- **01.09 AUDIO**: Detail the new audio files and their paths.
- **01.10 STRUCTURE**: Update the nesting hierarchy if needed.

## 🔊 ASSET PIPELINE
When adding new audio files:
1.  **Format**: Mono, 44.1kHz, MP3 (highly optimized).
2.  **Normalization**: Ensure peak volume is consistent with existing files (use the 300ms fade-in in `helpers.js`).
3.  **Naming**: Follow the folder convention (e.g., `mera_sansar/space/space_earth.mp3`).
4.  **Verification**: Run the audit script (if available) to ensure no duplicate IDs.

## 🆘 TROUBLESHOOTING FAQ

### Q1: Why isn't my new category appearing?
- **Check `router.js`**: Verify that the category key is added to the correct `showXCategory` function (Step 2, 3, or 4).
- **Check Icons**: Ensure the category key has a mapping in `finishSetup`'s `iconMap`.
- **Verify HTML**: Does the category require a new button in `index.html`, or is it dynamically generated?

### Q2: Why is the audio path failing?
- **Check `01.09`**: Look up the path pattern for your category.
- **Check Case Sensitivity**: Category keys must match folder names exactly (e.g., `varnamala` not `Varnamala`).
- **Verify Base Path**: Ensure `window.AUDIO_BASE_PATH` is correctly set in `init.js`.

### Q3: Why can't I click anything during playback?
- **Safety Lock**: The `ChildSafetyLock` is designed to block interaction while audio plays.
- **Developer Bypass**: Use `Ctrl+Alt+Shift+D` to toggle safety features for testing.
