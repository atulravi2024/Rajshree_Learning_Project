## 🏗️ ASSET ROLES & ARCHITECTURE

The project leverages a "Vanilla-Hybrid" asset structure, where heavy media (Audio/Images) is matched to lightweight data (JS Objects) and triggered by specific UI hooks.

### 1. 🎙️ AUDIO ASSETS (Total: 1003 Files)
- **Role**: Provides the "Voice & Feedback" of the application.
- **Detailed Map**: See `[01.09_PROJECT_AUDIO_MAPPING.md]` for granular paths.
- **Primary Categories**:
  - **Learning (Step 5)**: 933+ files for flashcards.
  - **Navigation (Step 1-4)**: Category intros and button feedback.
  - **System**: Quiz results, errors, and success alerts.

### 2. 🎨 ICON ASSETS (Total: ~550+ Points)
- **Role**: Visual cues for navigation and category identification.
- **Detailed Map**: See `[01.08_PROJECT_ICONS.md]` for per-screen mapping.
- **Types**:
  - **Unicode Emojis**: Primary source for cards and navbar.
  - **Custom SVGs**: Used for specialty UI elements (arrows, progress).
  - **Image PNGs**: Used for "Empty/Khali" states and branding.

### 3. 📊 DATA MODULES (Total: 52 Modules)
- **Role**: The "Brain" of the library; links text to audio/icons.
- **Location**: `frontend/src/js/data/`
- **Loaded In**: `index.html` via script tags.
- **Registry**: See `[01.14_GLOBAL_REGISTRY.md]` for variable mapping.
