## 🎙️ AUDIO ASSET NAMING
To ensure the `playSound()` engine works predictably, all MP3 files must follow this hierarchical pattern:

**Pattern**: `[domain]_[category]_[item].mp3`
- **Domain**: Top-level folder (e.g., `varnamala`, `mera_sansar`).
- **Category**: Sub-folder or group (e.g., `swar`, `animals`).
- **Item**: Specific word or identifier (e.g., `a`, `lion`).
- **Example**: `mera_sansar_animals_lion.mp3`

---

## 🎨 CSS & UI NAMING
We use a **Category-Prefix** system to ensure style isolation.

- **Class Pattern**: `.[feature]-[element]--[modifier]`
- **Examples**:
  - `.card-container--flipped`
  - `.navbar-icon--active`
  - `.overlay-btn--back`

---

## 📊 DATA VARIABLE NAMING
Variables in `01.14_GLOBAL_REGISTRY.md` must be all-caps for shared data.

- **Pattern**: `window.[DOMAIN]_[CATEGORY]_DATA`
- **Example**: `window.VARNAMALA_SWAR_DATA`
- **Purpose**: Prevents namespace collisions in the global `window` scope.

---

## 📁 FOLDER NAMING
- **Folders**: Lowercase with underscores (e.g., `mera_sansar`).
- **Memory Files**: Prefix with `01.XX` followed by ALL-CAPS (e.g., `01.20_NAMING_STANDARDS.md`).
