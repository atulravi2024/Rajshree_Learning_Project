## 🏰 THE "LOCAL SANCTUARY" PRINCIPLE
The application is designed as a secure, offline-first educational environment.
1. **Client-Only**: Zero external API calls for core functionality.
2. **Data Privacy**: No user data is transmitted to external servers.
3. **Immersive Lockdown**: The UI is designed to minimize distractions and accidental system navigation.

---

## 🛡️ SECURITY MECHANISMS

### 1. The Parental Gate (Time-Lock)
- **Logic**: Continuous 3-second hold.
- **Protection**: Guards `Settings`, `Developer Tools`, and `External Links`.
- **Implementation**: Managed by `ChildSafetyLock` state globally.

### 2. Interaction Throttling (Spam Guard)
- **Logic**: 400ms threshold between audio triggers.
- **Protection**: Prevents browser memory spikes and audio driver crashes from rapid clicking.

### 3. Browser State Persistence
- **Logic**: All settings (Volume, Layout, Safety) are persisted locally without cloud sync, ensuring 100% offline availability.

---

## 🛠️ DEVELOPER SECURITY BYPASS
For testing, developers can use a specific key sequence to bypass the Parental Gate.
- **Keys**: `Ctrl + Alt + Shift + D`
- **Visual**: A small lock icon 🔓 appears in the console.
