## 🚀 CATEGORY SCALING (100+ Modules)
To prevent `index.html` from becoming unmanageable, we utilize a "Dynamic Loader" philosophy.
1. **Lazy Loading**: Future iterations will move `Data Modules` into on-demand JSON fetching instead of pre-loaded script tags.
2. **Modular Registry**: `01.14_GLOBAL_REGISTRY.md` ensures central variable management even as the data layer expands.

---

## 🌍 MULTI-LANGUAGE READINESS (I18N)
The project is built to support Hindi as the primary language, with future-proofing for Marathi and English.

### 1. Data Layering
Data objects can be expanded without breaking the UI:
```javascript
{
  word: "पिता", 
  word_mr: "बाबा",   // Marathi layer
  word_en: "Father", // English layer
  icon: "👨",
  audio: "father"
}
```

### 2. UI Localization Strategy
- **Language Toggles**: A global `window.currentLanguage` variable will control which `word_XX` property is rendered by `display.js`.
- **Shared Assets**: Icons and generic "System Intros" (e.g., success sounds) are shared across all languages to minimize asset weight.
