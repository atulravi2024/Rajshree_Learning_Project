## 🏛️ PROJECT LINGUISTIC PILLARS

The Rajshree Learning Project operates on a strict "Hindi-First" vocabulary principle to ensure a culturally authentic and phonetically consistent learning experience for children aged 3-5.

### 1. Dual-Script Protocol
The system permits the use of two distinct scripts for content delivery, depending on the technical requirement:
- **Roman Script (Latin/Hinglish)**: Specifically used for Hinglish modes (e.g., `en-IN-NeerjaExpressiveNeural`) and technical metadata (JSON keys, filenames).
- **Devanagari Script (Native)**: Specifically used for formal Hindi modes (e.g., `hi-IN-SwaraNeural`) and instructional UI text.

### 2. Vocabulary Purity (The "Zero-Pure-English" Policy)
**CRITICAL**: "Pure English" words are strictly prohibited, regardless of the script being used. Vocabulary must always be Hindi-based.

- **Forbidden**: English words like "Mango", "Blue", "Monday", "Apple", "Table", "Rain".
- **Mandatory**: Hindi equivalents like "Aam", "Neela", "Somvar", "Seb", "Mez", "Baarish".
- **Rule Enforcement**: Even writing an English word in Devanagari (e.g., `मैंगो`) or Roman (e.g., `Mango`) is a violation of this standard.

## 🎙️ AUDIO SCRIPT GENERATION STANDARDS

To achieve high-fidelity and engaging audio, all script generation must adhere to the following standards:

### 1. Script Structural Attributes
- **Duration**: Every generated script must be between **20-30 seconds** long.
- **Tone**: Must be **funny, engaging, and playful**, while maintaining a **learning curve** appropriate for children aged 3-5.
- **Format**: Each script entry must follow the standard block format (ID, Time, File, Model, Script) for auditability.

### 2. Hybrid Voice Model Strategy
Category-specific generation requires choosing the appropriate persona from the [01.07_AUDIO_PLAYBACK_STANDARDS.md]:

| Voice Model | Language Script | Use Case / Persona |
| :--- | :--- | :--- |
| **Swara** (`hi-IN`) | Devanagari | **Basic/Instructional**: Formal teacher-like tone for basic concepts. |
| **Neerja** (`en-IN`) | Hinglish (Roman) | **Funny/Expressive**: Enthusiastic, cartoon-like engagement for kids. |
| **Madhur** (`hi-IN`) | Devanagari | **Parent Mode**: Calm, reliable adult voice for settings and safety gates. |

### 3. Generation Workflow (The "How" & "Where")
- **Source Location**: All source scripts reside in `memory/09 Audio Script/`.
- **Output Mapping**: Synced directly to `frontend/assets/audio/<category>/`.
- **Integrity**: Scripts must refer to the correct category folder to maintain manifest synchronization.

## 🗺️ UNIVERSAL SCOPE OF APPLICATION

These standards are universal and must be followed **EVERYWHERE** in the project:

| Layer | Application Details |
| :--- | :--- |
| **Audio Scripts** | All text strings for TTS generation must follow the duration/vocab rules. |
| **Text (UI/UX)** | All buttons, headers, and instructional labels in Devanagari or Hinglish. |
| **Images & Icons** | Assets representing objects (e.g., a "Clock") must not use English labels. |
| **Numbers** | Numerical representation must match the Hindi/Hinglish pedagogical style. |
| **Metadata** | Folder names, file names, and manifest properties. |
