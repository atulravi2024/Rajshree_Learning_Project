13: ## 🏗️ MODULAR ARCHITECTURE
14: The settings interface has been completely refactored from a simple modal into a modular, role-based dashboard.
15: - **Core Files**: 
16:   - `frontend/public/settings/settings.html`: The multi-section layout.
17:   - `frontend/public/settings/settings.css`: Premium glassmorphic styling.
18:   - `frontend/public/settings/settings.js`: Logic for section switching, volume control, and preference persistence.
19: - **Implementation**: The main app loads `settings.html` inside a secure `<iframe>` via `settings-modal.js`.
20:

23: ## 🔐 ROLE-BASED ACCESS
24: The interface is divided into specialized sections to ensure child safety and developer control.
25: 
26: | Section | Target Audience | Key Features |
27: |:---|:---|:---|
28: | **General** | All Users | Interface Theme, Language, System Volume. |
29: | **Kids** | Children (3-5) | Favorite Colors, Pet Selection, Reward Sounds. |
30: | **Parent** | Guardians | Safety Mode, Time Limits, Progress Reports. |
31: | **Admin** | Supervisors | User Management, Content Moderation. |
32: | **Developer** | Engineers | Internal Audit Link, Debug Console, Neural States. |

36: ## 🛡️ PARENTAL GATE & SAFETY
37: - **Access Hold**: Opening the settings requires a **3-second continuous press** on the gear icon.
38: - **Safety Toggle**: Enabling "Safety Mode" hides advanced settings and locks navigation.
39: - **Audio Feedback**: All settings interactions play kid-friendly sound effects.
40:

43: ## 📊 PREFERENCE PERSISTENCE
44: - **Mechanism**: `localStorage` is used to store user selections (volume, theme, selected kids' preferences).
45: - **Sync**: The `settings.js` script sends `postMessage` events to the parent window to update the global `window.RAJSHREE_CONFIG` in real-time.
