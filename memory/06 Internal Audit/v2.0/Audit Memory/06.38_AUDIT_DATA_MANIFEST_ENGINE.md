## 🏛️ ARCHITECTURAL DATA FLOW

The "Database" of the Frontier Protocol v2.0 is comprised of distributed JS-based manifests. This architecture ensures high-speed, build-less data access while maintaining a strict schema.

### 1. The Asset Manifest (`asset_manifest.js`)
- **Nature**: Automatically generated 740KB JS file.
- **Primary Export**: `window.PROJECT_ASSETS` (Array of Objects).
- **Metadata Schema**:
  - `name / path`: File identifiers and relative locations.
  - `type`: Categorization (doc, data, audio, visual, animation, video).
  - `risk`: Numerical risk score (1-100) used for sorting.
  - `compliance`: Binary compliance flag.
  - `perf`: Performance efficiency score.
  - `use / purpose`: Contextual strings used by the `assets_details.js` engine.

### 2. The Dashboard Registry (`DOMAIN_DATA`)
- **Nature**: Hardcoded structural registry in `dashboard.js`.
- **Function**: Defines the 12 neural slots (00-11) that form the core of the Internal Audit Hub.
- **Telemetry**: Contains `spark` arrays for Chart.js and `footerMetrics` for KPI rendering.

## ⚙️ DATA INTEGRITY & KPI LOGIC

### 1. Real-time Jitter Simulation
The `initDataRefresh()` engine in `dashboard.js` simulates live telemetry by applying randomized jitter to the `domain-kpi` elements.
- **Frequency**: Controlled by `window.CONFIG.refreshRate`.
- **Persistence**: Original values are stored in `data-base` attributes to prevent long-term value drift beyond acceptable thresholds.

### 2. Risk & Compliance Aggregation
Audit files like `06.35` (Assets) derive their "Fidelity Score" by aggregating the `risk` and `compliance` tags from the manifest.
- **Criticality**: Assets marked with `risk > 15` or `compliance: 0` are prioritized in the default sort order.

## 🚀 SCALING & EXPANSION

### 1. Global Geospatial Datasets
The map module (`map/js/map_data.js`) integrates with 180+ individual country files in `map/country/`. This "fragmented database" approach allows the Three.js globe to load only the necessary geospatial strings (lat/long) for the active sector, preventing memory overflow.

### 2. Static JSON Wrappers
Legacy data (e.g., `india_exhaustive.json`) acts as an "Archive Tier", referenced by the core scripts for deep-historical audits.
