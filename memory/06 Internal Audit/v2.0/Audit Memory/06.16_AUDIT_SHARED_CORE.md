## 🏗️ SYSTEM ARCHITECTURE

The **Shared Core** provides the foundational logic for all Frontier modules (Dashboard, Map, Terminal, Assets). It ensures visual consistency and state persistence across page transitions.

### 1. Global Configuration (`window.CONFIG`)
- **Action**: Centralized object for all user-customizable parameters.
- **Key Keys**:
  - `theme`: `frontier` (Standard), `obsidian`, `ghost`, etc.
  - `glowIntensity`: `0-100` (Controls `--neural-glow-opacity`).
  - `refreshRate`: `0.5-5.0` (Controls `initDataRefresh` interval).
  - `protocolLock`: `true/false` (Enables global grayscale filter and pointer-event blocking).

### 2. Settings Engine (`settings.js`)
- **Injection Node**: `#settings-sidebar-container`.
- **Accordion Logic**: Exclusive-open items with `hapticPulse()` feedback.
- **Effect Application**: `applySettingEffect(id, value)` maps UI input directly to `:root` CSS variables.
- **Volume Indicator**: 10-bar dynamic visualization based on `masterVolume`.

---

## ⚙️ LOGIC FLOW (Step-by-Step)

### Step 1: Component Initialization
1. `DOMContentLoaded` triggers `initSettings()` and `initNavbar()`.
2. Shared HTML templates (Accordions, Nav Links) are injected into the DOM.
3. `lucide.createIcons()` is called after injection to hydrate SVG elements.

### Step 2: Global State Sync
1. **Direct Application**: Slider values are applied to CSS variables via `root.style.setProperty`.
2. **Cross-Module Notification**: Functions like `updateVisualSettings()` or `initDataRefresh()` are called if present in the global scope to alert sub-modules of configuration changes.
3. **Auditor Identity**: `auditor-name` updates cause a `.name-updated` CSS animation pulse on the navbar name field.

### Step 3: Visual Protocol Overrides
- **Theme Switching**: Removes all `theme-*` classes from `body` before applying the new selection.
- **Chromic Aberration**: Toggles `.chroma-distort` class on `body` based on the intensity threshold.

## 🤖 AI OVERSIGHT INSTRUCTIONS
1. **Lucide Guard**: Always ensure `lucide.createIcons()` is called after any `innerHTML` modification.
2. **Selector Safety**: Use `querySelector` with specific container scopes (e.g., `container.querySelectorAll('.item')`) to avoid collisions with main page content.
3. **State Persistence**: The current implementation stores state in `window.CONFIG`. For multi-session persistence, check for `localStorage` integration (Planned for v2.1).
