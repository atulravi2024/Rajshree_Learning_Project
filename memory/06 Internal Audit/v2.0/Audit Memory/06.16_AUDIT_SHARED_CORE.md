## 🏗️ SYSTEM ARCHITECTURE

The **Shared Core** provides the foundational logic for all Frontier modules (Dashboard, Map, Terminal, Assets). It ensures visual consistency and state persistence across page transitions.

### 1. Persistent State (`visual_sync.js`)
- **Persistence**: Uses `frontier_config_v2` in `localStorage` for multi-session parity.
- **Global Sync**: Listens for the `storage` event to ensure visual synchronization across multiple hub modules (Dashboard, Map, etc.).
- **Master Knob**: `window.saveAndSyncVisuals()` handles both storage and immediate CSS root variable updates.

### 2. Atmospheric Injection (`background.js`)
- **Injection Node**: Appends `.background-container` to the `body` if absent.
- **Layer Logic**:
  1. `.background-grid`: 55px structural neon grid.
  2. `.holographic-glare`: Atmospheric illumination and depth.
  3. `.scanline-overlay`: 4px interval horizontal CRT simulation.
  4. `.crashing-noise`: Analog micro-grain grain for "fidelity" texture.

### 3. Notification Hub (`notifications.js`)
- **Central Modal**: `#alerts-modal` features a configurable `--glass-blur` (default 10px) and category-based filtering (SYSTEM, SECURITY, AUDITS, etc.).
- **Global State**: `NOTIFICATIONS` array serves as the single source of truth for the bell dropdown and the central hub.
- **Badge Sync**: `updateNotifBadge()` synchronizes unread counts across all active navigation layers.

## ⚙️ LOGIC FLOW (Step-by-Step)

### Step 1: Sequential Bootup
1. `visual_sync.js` (First Priority): Loads config and applies root variables to prevent "FOUC" (Flash of Unstyled Glow).
2. `background.js` (Second Priority): Dynamic injection of the 4 cinematic layers.
3. `notifications.js`: Preparing the dropdown and modal structures.
4. `lucide.createIcons()`: hydrates all dynamically injected elements.

### Step 2: Global Effect Propagation
1. **Trigger**: User modifies a config slider -> `saveAndSyncVisuals()` is executed.
2. **Persistence**: `localStorage` is updated instantly.
3. **Reflow**: `applyGlobalVisuals()` recalculates CSS variables like `--glow-intensity` and `--glass-blur`.
4. **3D Hook**: Sub-modules (e.g., Map `SceneShaders`) detect variable shifts and update Three.js properties in the animation loop.
