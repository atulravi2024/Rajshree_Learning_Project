## 🎨 DOM-PROXY ICON EXTRACTION (`createIconTexture`)

The Three.js scene uses a custom "DOM-Proxy" technique to inject 2D SVG icons (from Lucide) into the 3D scene as high-fidelity textures.

### 1. The Proxy Bridge
1. **Creation**: Creates an off-screen `120x120` HTML5 Canvas and a hidden `proxyDiv`.
2. **Lucide Injection**: Injects `<i data-lucide="${iconName}"></i>` into the `proxyDiv` and invokes `lucide.createIcons()` with the `scope: proxyDiv` parameter.
3. **SVG Extraction**: Reads the rendered `svg.outerHTML` from the `proxyDiv`.

### 2. Texture Serialization
1. **SVG-to-Blob**: Converts the SVG string into a `Blob({ type: 'image/svg+xml' })`.
2. **Image Loading**: Generates a `URL.createObjectURL` and loads it into a Javascript `Image` object.
3. **Canvas Drawing**: Draws the loaded `Image` onto the pre-created Canvas, adding a radial gradient glow behind the icon.
4. **THREE.js Sync**: Converts the final Canvas into a `THREE.CanvasTexture` for usage on globe sprites.

---

## 🛰️ SPRITE INTEGRATION

1. **Sprite Creation**: Uses the generated texture on a `THREE.SpriteMaterial`.
2. **Visual Properties**:
    - `transparent: true`.
    - `blending: THREE.AdditiveBlending`.
    - `depthWrite: false` (to prevent artifacts with atmospheric layers).
3. **Scaling**: Fixed `30x30` scale to preserve readability from distance.

### Cleanup Protocol
- **URL Revocation**: Immediately calls `URL.revokeObjectURL(url)` after texture load to prevent memory leaks in long-running audit sessions.
