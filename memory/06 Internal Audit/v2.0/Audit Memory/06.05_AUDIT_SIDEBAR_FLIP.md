## 🏗️ 3D STACKING ARCHITECTURE

### The Container Layer
- **Class**: `.neural-console`
- **Property**: `perspective: 1200px`
- **Function**: Establish the 3D space for the inner console to rotate within.

### The Animation Core
- **Class**: `.console-body`
- **Property**: `transition: transform 0.6s cubic-bezier(0.4, 0, 0.2, 1)`
- **Property**: `transform-style: preserve-3d`
- **Trigger**: The class `.flipped` is applied to this element via JavaScript.

### The Faces (Front & Back)
1. **Front (`.console-front`)**: Displays the standard Domain Overview (KPIs, Bar/Line Charts).
2. **Back (`.console-back`)**: Displays the "Detailed Audit" view (Resource lists, specific fidelity leaks).
   - **Critical CSS**: Must have `backface-visibility: hidden` and `transform: rotateY(180deg)`.

---

## ⚙️ ACTIVATION SEQUENCE

1. **User Interaction**: Click the `card-icon` (Sentinel) located in the `.console-header`.
2. **JS Event**: `toggleSidebarFlip()` is invoked.
3. **State Change**: `body.classList.toggle('flipped')`. (where body is the element with id console-body)
4. **Icon Sync**: Since the flip logic uses absolute positioning for faces, the header icon remains common and functional on both sides.

---

## 🤖 AI DIRECTIVES
- **Initialization**: On every `selectDomain()` call, the `.flipped` class MUST be removed to ensure the user starts on the "Front" face.
- **Icon Re-init**: Calling `lucide.createIcons()` after a flip is recommended to ensure any new `<i>` tags on the back face are rendered.
