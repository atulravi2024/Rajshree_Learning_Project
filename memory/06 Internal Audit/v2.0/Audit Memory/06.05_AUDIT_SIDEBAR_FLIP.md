## 🏗️ 3D STACKING ARCHITECTURE

### The Stage Layer
- **Container**: `#console-body`
- **Properties**: 
  - `perspective: 1200px` (HUD-standard depth).
  - `transform-style: preserve-3d`.
- **Function**: Establishes the coordinate system for the dual-face rotation.

### The Animation Core
- **Class**: `.console-body` (Target for the `.flipped` toggle).
- **Transitions**: `transform 0.8s cubic-bezier(0.4, 0, 0.2, 1)`.
- **Trigger**: Class addition to the wrapper element via `toggleSidebarFlip()`.

### The Faces (Front & Back)
1. **Front (`.console-front`)**: Primary overview face.
   - **Constraint**: `backface-visibility: hidden`.
2. **Back (`.console-back`)**: Semantic audit face.
   - **Properties**: 
     - `backface-visibility: hidden`.
     - `transform: rotateY(180deg)`.
   - **Content**: Detailed resource logs and fidelity leak indicators.

---

## ⚙️ ACTIVATION SEQUENCE

1. **User Interaction**: Click the `#console-icon-wrap` (Sentinel icon) in the header.
2. **Settings Lock**: Logic verifies that the sidebar is NOT in `settings` mode (which uses a flat overlay).
3. **JS Execution**: `toggleSidebarFlip()` toggles the `.flipped` class on `#console-body`.
4. **State Reset**: `selectDomain()` automatically removes `.flipped` to ensure new selections start on the Front face.
5. **Icon Refresh**: `lucide.createIcons()` is called post-flip to hydrate any dynamic markers on the back face.

---

## 🤖 AI DIRECTIVES
- **Initialization**: On every `selectDomain()` call, the `.flipped` class MUST be removed to ensure the user starts on the "Front" face.
- **Icon Re-init**: Calling `lucide.createIcons()` after a flip is recommended to ensure any new `<i>` tags on the back face are rendered.
