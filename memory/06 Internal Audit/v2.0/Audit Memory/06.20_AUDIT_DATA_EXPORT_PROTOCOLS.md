## 🏗️ DATA AGGREGATION SEQUENCE (`btn-export`)

The export protocol is a high-fidelity data extraction process that compiles 18+ distinct system datasets into a unified JSON schema for offline auditing.

### 1. Export Security & Constraints
- **Gatekeeper**: `if (window._isLockedDown) return;` (Export is disabled during Lockdown).
- **Mode Conflict**: `if (window._manualSearchToggle)` (Export is only available while in **Audit Mode**).
- **Notification**: UI notification `SYSTEM_MODE_CONFLICT` or alert triggered on unauthorized access.

### 2. Comprehensive Data Schema
The aggregation engine (`reportData`) captures the following keys:
| Source Category | Key | Technical Origin |

| **Global Ops** | `globalMetrics` | `window.DEFAULT_GLOBAL_METRICS`. |

## 📁 EXPORT TRANSACTION

1. **Serialization**: `JSON.stringify(reportData, null, 2)`.
2. **Blob Generation**: `new Blob([jsonStr], { type: 'application/json' })`.
3. **Trigger**: Creates ephemeral `<a>` element, sets `a.download` to `audit_report_frontier_...json`, and invokes `a.click()`.
4. **Log**: "[AUDIT_REPORT_EXPORTED] Successfully compiled all real-time datasets."

## 🏎️ SEARCH HISTORY PERSISTENCE (`recallSearch`)

The protocol maintains a "Quantum Search" history to allow auditors to re-examine suspicious routes across sessions.

### 1. Persistence Logic
- **Storage**: `localStorage` using the key `FRONTIER_SEARCH_HISTORY`.
- **Initialization**: Hydrated from the `window.SEARCH_HISTORY` array.
- **Recall Mechanism**: `recallSearch(index)` (in `map_modals.js`) restores inputs to `#map-search-from/via/to`, resets modes, and automatically triggers `#btn-run-search` after a 300ms delay.

### 2. UI Deployment
- **Interface**: Managed in `populateSearchHistoryList()`.
- **Control Overlay**: Includes a "CLEAR ALL RECORDS" button that purges both the local array and session storage.
