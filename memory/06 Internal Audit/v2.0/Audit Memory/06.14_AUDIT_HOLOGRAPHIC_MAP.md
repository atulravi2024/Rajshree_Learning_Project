## 🏛️ ARCHITECTURE OVERVIEW

The **Frontier Holographic Map** is a high-fidelity Three.js-based 3D visualization engine used for global audit tracking. It transition between a High-Fidelity (HF) photorealistic Earth and a stylized low-poly Wireframe "Hologram".

### 1. The Core Scene
- **Renderer**: WebGLRenderer with `alpha: true` and `antialias: true`.
- **Camera**: `PerspectiveCamera` (45° FOV) with distance-based LOD (Level of Detail) logic.
- **Group Hierarchy**: `globeGroup` contains the main Earth mesh, clouds, and atmosphere.
- **Lighting Dynamics**:
  - `SunLight`: Directional (0xffeedd, 0.8) positioned at (250, 80, 150).
  - `FillLight`: Directional (0x112244, 0.15) on the opposite side.
  - `AmbientLight`: Low-intensity (0.25) base logic.

### 2. Physical Components
- **High-Fidelity Mesh**: 90-unit radius sphere with 64x64 segments.
- **Holographic Grid**: 90.1-unit radius wireframe sphere (64x48 segments) for "Scanning" mode.
- **Textures**: NASA Blue Marble (Day), Night Lights (Emissive), and Water Map (Specular).
- **Atmosphere**: Dual-layer additive blending meshes (radii 93 and 100) for edge-glow.
- **Atmospheric Opacity**: Controlled by `glow` setting: `0.25 * (val / 100)` for Layer 1.
- **Cloud Layer**: Semi-transparent sphere (radius 91.5) with `0.00008` rotational drift.

## ⚙️ LOGIC & ANIMATION (Step-by-Step)

### Step 1: Initialization
1. Select target container (`#threejs-container`).
2. Load all 4 textures (Day, Night, Specular, Clouds).
3. Initialize `OrbitControls` with strict damping and zoom limits.

### Step 2: Node & Data Injection
1. Map `NODE_DATA` (cities/stations) to 3D space using `calculatePositionFromCoords(lat, lon)`.
2. Inject `Sprite` markers (Cyan for Stable, Red for Critical).
3. **Audit Feature**: Selected nodes oscillate in scale using `Math.sin(Date.now() * 0.008)`.

### Step 3: Animation Loop (`animate()`)
1. **Clock Sync**: Update all time-based animations (satellites, pulses).
2. **Smooth Zoom**: Interpolate camera position towards `window._mapTargetCameraZ`.
3. **Globe Drift**: Apply subtle rotational forces to the cloud layer.
4. **LOD Management**: High-altitude (wireframe) vs. Low-altitude (HF texture) switching.

## 🤖 AI OVERSIGHT INSTRUCTIONS
1. **Performance**: Always ensure `renderer.setPixelRatio(window.devicePixelRatio)` is called to prevent blur on Retine/4K displays.
2. **Texture Loading**: Textures are loaded from CDNs. If a network leak occurs, check for fallback colors (`0x112233` for globe, `0xffffff` for clouds).
3. **Coordinate Math**: When adding new locations, use the standard formula:
   ```javascript
   x = radius * Math.cos(phi) * Math.cos(theta);
   y = radius * Math.sin(phi);
   z = radius * Math.cos(phi) * Math.sin(theta);
   ```
