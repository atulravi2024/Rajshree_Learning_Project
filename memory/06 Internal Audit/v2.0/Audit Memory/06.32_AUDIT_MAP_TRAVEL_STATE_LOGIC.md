## 🧭 TRAVEL OPERATIONS (`map_travel_settings.js`)

The navigation system features a multi-tiered travel engine for varying audit granularities.

### 1. Pointer Mode Management
- **Registry**: `circle` (Default), `plane`, `jet`, `rocket`, `satellite`, `drone`, `helicopter`, `ship`, `submarine`, `car`, `train`.
- **Logic**: 
    - Selection updates `window.SELECTED_POINTER_ICON`.
    - Triggers `window.createIconTexture` for high-fidelity sprite re-generation.
    - Updates local speed constant for duration calculations.

### 2. Altitude Control
- **Logic**: Toggles between `auto`, `low`, `mid`, and `high` orbital layers.
- **Math**: Adjusts the `THREE.Sprite` scale and Y-offset distance from the globe's surface (Offset range: 0.1 to 10.0 units).

## ⚙️ VIEWPORT STATE MAINTENANCE (`map_state.js`)

The audit map maintains a persistent internal state to prevent visual desynchronization during high-frequency telemetry.

- **Global Flags**:
    - `_mapGlobeDesign`: High-Fidelity vs. Wireframe.
    - `_mapViewMode`: Satellite vs. Atlas.
    - `_mapThemeMode`: Light vs. Dark.
    - `_mapAtmosphere`: Boolean.
    - `_mapSunlight`: Boolean.
- **Sync Logic**: Each state change is mirrored in the local `window.STATE` object to ensure consistent "Deep Scan" refreshes.

## 🎮 THREE.JS CONTROL ARCHITECTURE (`map_controls.js`)

Customized environment interaction for the Lead Auditor.

- **OrbitControls Protocol**:
    - `enableDamping`: true (Smoothness factor: 0.05).
    - `minDistance`: 30 (Surface proximity lock).
    - `maxDistance`: 550 (Atmospheric exit lock).
    - `autoRotate`: Controlled via the UI `btn-rotation` toggle.
- **Interaction Constraints**: Pan and tilt are restricted during active "Quantum Search" to preserve route focus.
