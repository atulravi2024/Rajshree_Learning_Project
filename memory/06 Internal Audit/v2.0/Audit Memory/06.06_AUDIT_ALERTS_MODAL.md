## 🏗️ MODAL ARCHITECTURE

### Overlay & Container
- **Class**: `.alerts-modal-overlay`
- **Property**: `backdrop-filter: blur(10px)`
- **Sizing**: Fixed `650px` width and `600px` height to prevent layout shifts during filtering.

### Navigation & Filtering
- **Component**: `.modal-filter` (Fixed Height: `36px`).
- **UX**: Hidden scrollbar for a clean HUD look.
- **Logic**: JS filters `NOTIFICATIONS` and calls `renderModalAlerts()`.
- **Re-init**: `lucide.createIcons()` is called after every filter swap.

---

## ⚙️ BEHAVIORAL LOGIC

### Unread Indicators
- **Element**: `.alert-marker`
- **Visual**: Cyan glow (`box-shadow`) applied when `n.unread === true`.
- **Sync**: Linked to the navbar `.nav-badge`.

### Interactive Controls
1. **Mark Read/Unread**: Each item has a toggle button that updates the local state and triggers `updateNotifBadge()`.
2. **Empty States**: If a category has no alerts, the `.empty-alerts` view is injected with an informative Lucide icon.
3. **Outside Click**: Modal closes automatically if the background overlay is clicked (logic in `initNotifications`).
