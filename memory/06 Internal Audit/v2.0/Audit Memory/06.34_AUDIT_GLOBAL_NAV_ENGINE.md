### 1. Active Link Synchronization (`shared/navbar.js`)
The engine uses a deterministic filename-to-ID mapping to ensure accurate "Active Trace" indicators:
- **`dashboard.html`** → `nav-dashboard`
- **`global.html`** → `nav-terminal`
- **`map.html`** → `nav-map`
- **`assets.html`** → `nav-assets`
- **Logic**: Injects the `.active` class based on `window.location.pathname`.

### 2. Analysis vs. Search Toggle (`nav_master.js`)
Specifically for the Map domain, the nav engine orchestrates a two-face analysis pivot:
- **Front Side (Analysis)**: `_manualSearchToggle = false`. Displays all `.navbar-category` metrics.
- **Back Side (Search)**: `_manualSearchToggle = true`. Activates the `#global-search-container` and metrics bar.
- **3D Sync**: 
    - **Visual**: Toggles visibility of `_mapLinesGroup`, `_mapStatusNodesGroup`, and `_mapRegionGroup`.
    - **Zoom**: Automates camera depth to **150** (Search) or **250** (Analysis).

## 🔄 SIDEBAR CARD-FLIP ENGINE (`nav_back.js`, `nav_front.js`)

A centralized 3D state-machine used across all project domains to reveal detailed audit reports.

### 1. Front-to-Back Transition (`nav_back.js`)
- **Trigger**: Click on `.card-icon` (Dashboard) or `.nav-mode-trigger` (Map).
- **Mechanism**: 
    - Toggles `.flipped` on the `#console-body` or `.nav-side-card-inner`.
    - Forces a delayed `opacity: 1` on the back-face content to prevent layout z-fighting during rotation.

### 2. Parity Rules (`nav_front.js`)
- **Logic**: 
    - Ensures that any "Close" or "Discard" action on a modal or sidebar automatically resets the flip state to the Front face.
    - Clears any temporary `.flipped` classes on the `DOMContentLoaded` event of the next page load.
