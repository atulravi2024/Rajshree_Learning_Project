## 🎨 ATMOSPHERIC RAYLEIGH SCATTERING

The Frontier Map utilizes a dual-layer shader to simulate a high-fidelity planetary atmosphere.

### 1. Primary Glow Layer
- **Shader Logic**: `AdditiveBlending` with a fresnel-based falloff.
- **Color Parameters**: `0x00f0ff` (Cyan) / `0xffffff` (White fallback).
- **Emissive Math**: `window._mapGlobeMat.emissiveIntensity` ranges from `0.0` (Offline) to `0.9` (Boosted).
- **Scale Offset**: The atmospheric shell is scaled to `1.02x` the globe radius to prevent Z-fighting.

### 2. Physical Constants
- **Cloud Parallax**: Cloud meshes rotate independently at `0.0001 rad/frame` on the Y-axis.
- **Lighting**: 
    - **Sunlight**: Directional (0xffffff, 1.25) casting soft shadows.
    - **Ambient**: Low-intensity (0.25) to preserve holographic contrast.

## 🚀 VISUAL BOOST (`applyVisualBoostState`)

A dedicated graphics enhancement engine that shifts shader parameters in real-time.

### Dynamic Interaction Logic
1. **High-Fidelity (HF) Mode**:
    - `Sunlight`: 0.8 -> 1.45 intensity.
    - `Emissive`: 0.7 -> 1.0 peak.
    - `Specularity`: Bump scale increased to `0.75` for terrain detail.
2. **Wireframe (HF) Mode**:
    - `Sunlight`: Suppressed (0.0).
    - `Grid Opacity`: 0.35 -> 0.65.
    - `Glow Scale`: Aura expands to `255` units on the mesh.

## 🔄 ROTATION KINETICS

Holographic rotation is managed through two distinct mechanisms:
- **Orbital Controls**: Continuous auto-rotation via `controls.autoRotate = true`.
- **GSAP-Like Easing (`gsap_like_rotate`)**: 
    - Used for mode transitions (e.g., Monitoring -> Threat).
    - **Logic**: Iterative rotation updates with sinusoidal deceleration (`Math.sin` or `lerp`).
    - **Targeting**: Snaps to specific degree orientations for primary audit sectors.
