## 🎨 ATMOSPHERIC RAYLEIGH SCATTERING

The Frontier Map utilizes a dual-layer shader to simulate a high-fidelity planetary atmosphere.

### 1. Primary Glow Layer
- **Shader Logic**: `AdditiveBlending` using `MeshLambertMaterial` for the dual-shell atmosphere.
- **Color Parameters**: `0x88ccff` (Cyan-Blue) for the inner shell, `0x4488ff` for the outer shell.
- **Emissive Math**: `window._mapGlobeMat.emissiveIntensity` scales by `glowIntensity` (0.0 to 1.0) and theme defaults (e.g., 0.7 for Dark mode).
- **Physical Radii**: 
    - **Atmosphere 1**: 93r (Backside rendering for halo effect).
    - **Atmosphere 2**: 100r (Backside rendering for deep space glow).

### 2. Physical Constants
- **Cloud Parallax**: Cloud meshes rotate independently at `0.00008 rad/frame` on the Y-axis.
- **Lighting Dynamics**: 
    - **Sunlight**: Directional (0xffeedd, 0.8) at (250, 80, 150).
    - **Ambient**: 0.25 intensity (Color: `0x223344`) for contrast parity.

## 🚀 VISUAL BOOST (`applyVisualBoostState`)

A dedicated graphics enhancement engine that shifts shader parameters in real-time.

### Dynamic Interaction Logic
1. **High-Fidelity (HF) Mode**:
    - `Sunlight`: 0.8 -> 1.45 intensity peak.
    - `Emissive`: 0.7 -> 1.0 peak (Dark Theme).
    - `Bump Scaling`: Increased to `0.75` for tactile terrain fidelity.
2. **Wireframe Mode**:
    - `Sunlight`: Suppressed (0.0) for pure holographic look.
    - `Grid Opacity`: 0.35 -> 0.65 boost.
    - `Aura Expansion`: Scale increases to `255r` with `0.45` opacity.

## 🔄 ROTATION KINETICS

Holographic rotation is managed through two distinct mechanisms:
- **Orbital Controls**: Continuous auto-rotation via `controls.autoRotate = true`.
- **GSAP-Like Easing (`gsap_like_rotate`)**: 
    - Used for mode transitions (e.g., Monitoring -> Threat).
    - **Logic**: Iterative rotation updates with sinusoidal deceleration (`Math.sin` or `lerp`).
    - **Targeting**: Snaps to specific degree orientations for primary audit sectors.
