## 🏛️ ARCHITECTURAL OVERVIEW

The Terminal Interface in v2.0 provides a low-level "Lead Auditor" entry point into the Frontier Protocol. It is split into **Global** (System-wide) and **Domain** (Sector-specific) command processors.

### 1. Theme & Visual Persistence
The terminal supports a **10-tier** thematic layer registry, persisted via `localStorage` (`fp_theme`):
- **Registry**: `frontier` (Default), `obsidian`, `ghost`, `nordic`, `synth`, `royal`, `emerald`, `crimson`, `solar`, `stealth`.
- **Mechanism**: Toggles the `theme-{name}` class on `document.body`.

### 2. Perceptual Core (Audio & Animation)
- **Matrix Decrypting**: Lines use `requestAnimationFrame` with a 600ms duration and per-line cascading delays (180ms steps).
- **Synthesized Audio**: Uses zero-asset square/sine/sawtooth waves:
    - **Keystroke**: Square wave (1200Hz → 400Hz).
    - **Success**: Double sine chord (440Hz/660Hz).
    - **Error**: Sawtooth wave (180Hz → 60Hz).

### 2. Command Shell Engine
- **Command History**: Navigable history via `ArrowUp/ArrowDown` and Tab-autocomplete for power users.
- **Broadcast System**: High-priority message injection into the `broadcast-bar` HUD element across the system.

## ⚙️ LOGIC LAYERS

### 1. Command Registry (`global.js`)
The terminal supports 25+ operational commands, including:
- **`neural-sync`**: Visualizes sector-by-sector alignment percentages.
- **`sector-ping`**: Simulated packet exchange with specific domains.
- **`protocol-x`**: Classified sequence for accessing restricted memory layers.
- **`threat-level`**: Real-time aggregation of THREAT and WARNING states.

### 2. Sector Navigation (`sector-jump`)
- **Direct Binding**: Allows the Auditor to jump directly to a specific neural node (e.g., `sector-jump 06`).
- **Persistence**: Maintains the selected `THEME` and `AUDIT_DEPTH` via `localStorage`, ensuring a seamless transition from the 3D Dashboard to the 2D Terminal.
