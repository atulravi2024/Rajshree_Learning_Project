## 🏛️ ARCHITECTURAL OVERVIEW

The Notification System in v2.0 transitions from ad-hoc alerts to a **Centralized Neural Alerts Hub**. It is designed to be injected into any "Frontier" page via the shared `notifications.js` script.

### 1. Dual-Interface Delivery
- **Dropdown Overlay**: A high-speed preview accessed via the `notification-bell` in the navbar. Provides "Mark all as read" functionality and quick-toggles for unread status.
- **Central Alerts Hub (Modal)**: A deep-dive interface (`alerts-modal`) supporting category-specific filtering and exhaustive alert history.

### 2. Domain-Aware Categories
Alerts are classified across six critical audit vectors:
- **SYSTEM**: Core protocol upgrades and hardware latency shifts.
- **SECURITY**: Credential leaks and unauthorized access patterns.
- **AUDITS**: Fidelity deviations and sector drift reports.
- **REPORTS**: Compliance summaries and generated documentation.
- **MIGRATIONS**: Data sync status and legacy archive indexing.
- **ANALYTICS**: Traffic spikes and heuristic pattern mismatches.

---

## ⚙️ LOGIC & DATA ENGINE

### 1. The Alert Registry (`notifications.js`)
- **Data Model**: Uses a structured array of objects containing `id`, `type`, `title`, `desc`, `time`, `icon`, and `unread`.
- **Iconography**: Leverages the Lucide library for status-specific visual markers (e.g., `alert-triangle` for drift, `shield-alert` for security).

### 2. Reactive UI Lifecycle
- **Injection Logic**: `initNotifications()` uses `insertAdjacentHTML` to lazily inject the Modal and Dropdown elements if they are missing from the base HTML.
- **State Sync**: `updateNotifBadge()` calculates unread counts in real-time, updating the global navbar marker.
- **Filtering Logic**: `renderModalAlerts()` implements a simple but effective filter engine using array `filter()` against the `NOTIFICATIONS` constant.

---

## 🚀 PERSISTENCE PROTOCOL

Active notification states (Unread/Read) are managed in-memory during the session.
> [!NOTE]
> Future iterations (v3.0) propose migrating the `NOTIFICATIONS` array to `localStorage` via the `visual_sync.js` engine to maintain unread status across browser refreshes and tab switches.
