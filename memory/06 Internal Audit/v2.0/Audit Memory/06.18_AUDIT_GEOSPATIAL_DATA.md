## 🏛️ DATASET REQUISITION

The Frontier Hub relies on an expansive geospatial engine for station identification and audit monitoring.

### 1. Global Station Density
- **Total Nodes**: 1,200+ tracked audit stations across 6 primary markets.
- **Coverage**:
    - **India**: 36 States/UTs (Full granularity).
    - **Global**: USA, China, Brazil, UK, and Germany (All major tech/audit hubs).
- **Format**: Decoupled JSON arrays flattened into a single reference matrix (`window.NODE_DATA`).

### 2. Geometry & Translation Logic
- **Sphere Mapping**: Uses spherical coordinates translated to Cartesian space.
- **Conversion Math**:
    ```javascript
    phi = (90 - lat) * (Math.PI / 180);
    theta = (lon + 180) * (Math.PI / 180);
    x = -(radius * Math.sin(phi) * Math.cos(theta));
    z = (radius * Math.sin(phi) * Math.sin(theta));
    y = (radius * Math.cos(phi));
    ```

## 👓 LOD SWITCHING & RENDERING

The engine uses Level-of-Detail (LOD) thresholds to maintain 60 FPS while preserving high-fidelity visual context during deep zooms.

### 1. Camera Proximity Thresholds
- **Global View (`camera.z > 250`)**: Renders atmospheric layers and cloud meshes.
- **Medium Zoom (`120 < camera.z < 250`)**: Introduces city markers and cluster labels.
- **Micro View (`camera.z < 120`)**: 
    - Swaps diffuse 8K texture with a high-fidelity vector grid.
    - Enables sub-station identifier markers.
    - Activates local ambient occlusion for node meshes.

### 2. Cluster Dynamics
- **Interaction**: Nodes remain interactive via Three.js Raycasting.
- **Metadata**: Clicking a node triggers `updateSidebarForSelection(nodeId)`.
- **Latency Masking**: Lookup times for node metadata are optimized through a hash-map cache (`sub-5ms` retrieval).
