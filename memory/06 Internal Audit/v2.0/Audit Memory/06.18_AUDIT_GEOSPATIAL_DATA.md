## 🏛️ DATASET REQUISITION

The Frontier Hub relies on an expansive geospatial engine for global station identification and audit monitoring.

### 1. Global Station Density
- **Total Nodes**: 5,000+ tracked audit stations across 180+ sovereign nations.
- **Coverage**:
    - **India**: 36 States/UTs (Full granularity).
    - **Global**: 188 country-specific modules (e.g., `usa.js`, `china.js`, `uae.js`) containing city-level latitudes and longitudes.
- **Format**: Fragmented JS registries for memory efficiency. Only required country datasets are loaded into the active Three.js session buffer.


### 2. Geometry & Translation Logic
- **Sphere Mapping**: Standard 3D spherical-to-cartesian projection.
- **Conversion Math (Three.js standard)**:
    ```javascript
    const phi = (90 - data.lat) * (Math.PI / 180);
    const theta = (data.lon + 180) * (Math.PI / 180);
    const r = 90; // Globe Radius
    const x = -(r * Math.sin(phi) * Math.cos(theta));
    const z = (r * Math.sin(phi) * Math.sin(theta));
    const y = (r * Math.cos(phi));
    ```
- **Precision**: Nodes are positioned at index `r=90`, while labels (LOD) scale from `r=91.5` to `r=92` to prevent mesh Z-fighting.

## 👓 LOD SWITCHING & RENDERING

The engine uses Level-of-Detail (LOD) thresholds to maintain 60 FPS while preserving high-fidelity visual context during deep zooms.

### 1. Camera Proximity Thresholds
- **Global View (`dist > 180`)**: Renders atmospheric layers and cloud meshes. Fades region labels.
- **Medium Zoom (`120 < dist < 260`)**: Introduces city markers and cluster labels.
- **Micro View (`dist < 120`)**: 
    - Enables sub-station identifier markers.
    - Activates local ambient occlusion for node meshes.

### 2. Cluster Dynamics
- **Interaction**: Nodes remain interactive via Three.js Raycasting.
- **Metadata**: Clicking a node triggers `updateSidebarForSelection(nodeId)`.
- **Latency Masking**: Lookup times for node metadata are optimized through a hash-map cache (`sub-5ms` retrieval).
