## ⏺️ HIGH-FIDELITY VIDEO RECORDING (`map_record.js`)

The audit system includes a professional-grade screen capture engine for archiving globe animations.

### 1. WebCodecs Integration (MP4)
- **Primary Method**: Uses `VideoEncoder` and the `Mp4Muxer` library for localized MP4 generation.
- **Bitrate**: `4,000,000` (4MB/s) for high clarity.
- **Framerate**: Locked to `30 FPS`.
- **Canvas Capture**: Extracts the Three.js canvas buffer into a `VideoFrame` timestamped at `(encodedFrameCount * 1,000,000) / FRAME_RATE` microseconds.

### 2. MediaRecorder Fallback (WebM)
- **Fallback**: Switches to the browser's native `MediaRecorder` API if WebCodecs is unsupported.
- **Format**: `video/webm`.
- **Logic**: Uses a `proxyCanvas` to handle resolution scaling and background normalization (e.g., clearing the wireframe black-level).

### 3. File Serialization
- **Naming**: `frontier-globe-recording-[ISO-Timestamp].[EXT]`.
- **Cleanup**: Invokes `URL.revokeObjectURL` immediately after the auto-click download to prevent memory leaks during long-running audit sessions.

## 🏛️ MODAL ARCHITECTURE (`map_modals.js`)

Content management for the holographic audit popups.

### 1. Audit Log Modal (`populateLogsModal`)
- **Logic**: Filters and injects `window.AUDIT_LOG_HISTORY` entries into the modal body.
- **Styling**: Uses internal Lucide icons to indicate event types (Threat/System/User).

### 2. Sidebar Flip State-Machine
- **Control**: Managed via `#btn-view-logs` and `#btn-view-info`.
- **Transition**: Uses CSS 3D transforms (`transform: rotateY(180deg)`) on the `.nav-side-card-inner` container for a physical card-flip effect.
