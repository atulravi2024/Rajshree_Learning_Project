## 🎨 STYLING WORKFLOW (Step-by-Step)

### Step 1: Token Initialization (`:root`)
- **AI/Human Action**: Define `--cyan` (#00f0ff) as the primary illumination token.
- **Glow Hierarchy**: 
  - `subtle` (10%): Ambient background aura.
  - `soft` (20%): Standard element halo.
  - `mid` (40%): Active card and pulse emission.
  - `max` (95%): Peak CRT/Holographic flare.
- **Neon Balance**: Use `--cyan-glow` (0.25 alpha) for subtle halos and `--cyan-glow-strong` (0.5 alpha) for active states.
- **Dynamic Multipliers**: Glow values are dynamic: `rgba(var(--accent-rgb), calc(0.2 * var(--neural-glow-opacity)))`.

### Step 2: Cinematic Background Setup
1. **Void Gradient**: Apply a 3-layer radial/linear gradient blend for the deep space effect.
2. **Diagnostic Grid**: Set `background-size: 55px 55px` to match mockup density.
3. **Scanlines**: Apply a repeating linear gradient (0deg) with 4px intervals for the CRT effect.

### Step 3: Neural Console Glassmorphism
- **Step**: Apply `backdrop-filter: blur(12px)` and `border-left: 1px solid var(--border-glow)`.
- **Shadows**: Use `box-shadow: -4px 0 30px rgba(0, 0, 0, 0.5)` to create separation from the grid.

### Step 4: The 1.3x Launch Animation "Magic"
1. **Target**: `.btn-launch` inside the sidebar.
2. **Problem**: Default `overflow: auto` on `.console-body` clips the button when it scales.
3. **Step-by-Step Fix**:
   - Set `transition: overflow 0s` on parent.
   - Use `:has(.btn-launch:hover)` to toggle `overflow: visible` dynamically.
   - Ensure `min-height: 0` is set to prevent layout recalculation flicker.

### Step 5: Sentinel Pulsing & 3D Flip
1. **Animations**: 
   - `sentinel-aura`: Emerald green 4s cycle for `stable`.
   - `sentinel-glitch`: 0.15s high-frequency jitter for `active`.
   - `sentinel-spin`: 6s sustained rotation for `warning`.
   - `sentinel-alarm`: 0.5s urgent crimson pulse for `threat`.
   - `hapticPulse`: UI feedback animation for setting changes.
   - `glarePulse`: Holographic expansion on interaction.
2. **Interactive States**:
   - **Header Icon**: `scale(1.2) rotate(10deg)` on hover.
   - **Theme Ghost Fix**: Overrides card hover colors to maintain dark-on-light contrast.
   - **Data Masking**: `.data-masked .kpi-value` uses `filter: blur(var(--data-blur-amount))`.
   - **Chroma Switch**: `.chroma-distort` applies a 3-channel shift via `hue-rotate` and `sepia`.
3. **3D Perspective**:
   - `.console-body`: Uses `perspective: 1200px` and `transform-style: preserve-3d`.
   - `.flipped`: Applies `transform: rotateY(180deg)`.
4. **Theme Registry**:
   - `.theme-frontier`: Default Cyan HUD.
   - `.theme-obsidian`: Purple Dark-Mode.
   - `.theme-ghost`: Blue Light-Mode (with inverted contrast).
   - `.theme-nordic`: Frost Blue.
   - `.theme-synth`: Neon Pink.
   - `.theme-emerald`: Green Matrix-style.
   - `.theme-crimson`: Red Alert UI.
   - `.theme-solar`: Teal Solarized.
   - `.theme-stealth`: Carbon Grayscale.

## 🤖 AI DIRECTIVES
- **Transition Timing**: All interactive scale-ups (1.3x) MUST use `cubic-bezier(0.175, 0.885, 0.32, 1.255)` to prevent rigid, linear motion.
- **Fidelity Sync**: If a background grid size is changed in CSS, the `gridOpacity` logic in JS must be re-calibrated.
- **Animation Sync**: Verify that `wave-jump` animation-duration is randomized (1.2s to 1.8s) in JS to prevent synchronized "robotic" waves.
- **Shadow Performance**: Use `will-change: transform` on the launch button to maintain high FPS during the 1.3x scale-up.
- **Tooltip Layering**: Ensure `.sentinel-tooltip-float` has `z-index: 10000` to sit above all holographic glare and sidebar layers.
