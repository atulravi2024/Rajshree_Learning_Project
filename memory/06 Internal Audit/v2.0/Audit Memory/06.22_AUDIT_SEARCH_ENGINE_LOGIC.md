## 🗺️ LOCATION SEARCH (`searchDeepLocation`)

The search engine is built for "Anyway from Anyway" reliability, handling common name variations and nested data structures.

### 1. Alias & Synonym Redirection
- **Mechanism**: Checks `window.MAP_SEARCH_ALIASES` for alternate names before querying the database.
- **Registry**: 
    - `bombay` -> `mumbai`
    - `bangalore` -> `bengaluru`
    - `madras` -> `chennai`
    - `calcutta` -> `kolkata`
    - `pondy` -> `puducherry` (and 4+ others).

### 2. Matching Heirarchy
1. **Direct Match**: Checks `window.LOCATION_COORDS` for O(1) retrieval.
2. **Recursive Search**: Calls `findNestedCoords()` if searching within a country or state object.
3. **Fuzzy Retrieval**: If no exact match exists, collects all partial matches (`string.includes`) and returns the first result.

## ⌨️ AUTOCOMPLETE ENGINE

A real-time suggestion system designed for sub-ms feedback.

### Technical Parameters
- **Debounce**: 300ms delay to prevent excessive CPU load.
- **Trigger Depth**: Activates after 3 characters are typed.
- **Search Limit**: Constrained to the **Top 5** results to preserve UI performance.
- **Recursive Suggestions**: Iterates through `LOCATION_DATA[Country][State][City]` to build suggestion objects containing `{city, state, country, coords}`.

### UI Interaction
- **Rendering**: Dynamically injects `.suggestion-item` into `#map-search-suggestions`.
- **Sync**: Clicking a suggestion instantly calls `window.rotateGlobeToCoords()` and clears the suggestion list.
