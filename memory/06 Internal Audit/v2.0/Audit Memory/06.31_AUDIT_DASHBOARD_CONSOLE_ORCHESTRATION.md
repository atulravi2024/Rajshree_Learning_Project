## 🏛️ NEURAL CONSOLE STATE-MACHINE

The dashboard operates on a modular state-machine logic for transitioning between high-level project views and granular domain audits.

### 1. View Selection (`selectDomain`)
- **Mechanism**: Toggles the `data-mode="details"` attribute and ensures the `.frontier-main` layout class is persistent.
- **Activation**:
    - Clears all existing `.domain-card` active states.
    - Forces a 3-part update: **Header Title**, **Header Icon**, **Body HTML**.
- **Sync**: Invokes a 60ms delayed `renderSidebarChart` to prevent chart initialization before the DOM settles.

### 2. Physical Flip Animation (`toggleSidebarFlip`)
- **Mechanism**: Toggles the `.flipped` class on the `#console-body` container.
- **CSS Logic**: Uses `transform-style: preserve-3d` and `backface-visibility: hidden`.
- **Lucide Sync**: Forces a `lucide.createIcons()` call on every flip to re-render SVG icons on the hidden face.

## 📏 DYNAMIC HEADER SCALING (`scaleConsoleTitle`)

To accommodate long project names within the fixed 360px sidebar width, a rem-based iterative scaling algorithm is used.

- **Constraint**: `maxWidth = 280`.
- **Initial State**: `0.85rem`.
- **Logic Loop**: 
    - Continually decreases `currentSize` by `0.05rem` increments.
    - Stops when `title.scrollWidth <= 280` or `size < 0.5rem`.
- **Event Hook**: Triggered on `window.resize` and every `selectDomain` call.

## 📄 PATTERN BUILDER (HTML Templating)

### 1. Card Face Logic (`buildCardHTML`)
- **Mechanism**: JS template strings for `.card-top`, `.card-title`, and `.card-body`.

### 2. Console Face Logic (`buildSidebarHTML`)
- **Neural Logic**: Generates an "AI Interpretation" string based on the current sector status (Stable/Warning/Threat).
- **Template Logic**: Conditional mapping for System Metrics, Keywords, and Resource lists, each with localized collapsibility.
