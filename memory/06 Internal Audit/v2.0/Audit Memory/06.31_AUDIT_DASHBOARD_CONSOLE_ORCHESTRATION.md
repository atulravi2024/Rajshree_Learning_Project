## 🏛️ NEURAL CONSOLE STATE-MACHINE

The dashboard operates on a modular state-machine logic for transitioning between high-level project views and granular domain audits.

### 1. View Selection (`selectDomain`)
- **Mechanism**: Orchestrates the transition from the domain grid to the neural sidebar.
- **Activation Sequence**:
    1. Removes `.console-hidden` from the main container.
    2. Updates `#console-title` and `#console-icon-wrap`.
    3. Triggers `scaleConsoleTitle()` to normalize the header.
    4. Invokes `buildSidebarHTML(d)` and `renderSidebarChart(d)`.
- **Sync**: Automatically removes the `.flipped` class to ensure new selections start on the "Front" Overview face.

### 2. State Machine Lifecycle
- **Chart Management**: `renderSidebarChart(d)` MUST call `activeConsoleChart.destroy()` to prevent memory leaks and overlapping canvas layers.
- **Dynamic Content**: `buildSidebarHTML(d)` generates distinct HTML for the "Front" (Overview) and "Back" (Detailed Audit) faces based on the current sector's `DOMAIN_DATA`.
- **Flip Execution**: `toggleSidebarFlip()` toggles the `.flipped` class on `#console-body` (Perspective: 1200px).

## 📏 DYNAMIC HEADER SCALING (`scaleConsoleTitle`)

To accommodate long project names within the fixed 360px sidebar width, a rem-based iterative scaling algorithm is used.

- **Constraint**: `280px` maximum title width threshold (within fixed `360px` panel).
- **Initial State**: `0.85rem` font size.
- **Logic**:
    - Checks the `scrollWidth` of the title element.
    - If `width > 280`, decrements font-size by `0.05rem`.
    - **Minimum Threshold**: Stop scaling at `0.5rem` to maintain basic readability.
- **Trigger**: Executed on every `selectDomain` call and on `window.resize`.
