## ⚙️ EXECUTION FLOW (Step-by-Step)

### Step 1: State Initialization
- **Action**: Load `DOMAIN_DATA` structure. Each object must contain `slot`, `domainId`, and a `spark` array of historical data.
- **Card Type Registry**:
  - `chart-line`: Premium line chart with gradient fill.
  - `chart-bar`: Diagnostic bar chart for drift/latency.
  - `chart-shield`: Bar chart with increased shadow for system integrity.
  - `waveform`: Pure HTML/CSS animated sequence for audio domains.
- **Waveform Registry**: Populate `WAVE_HEIGHTS` (60 points) to drive the organic audio visualization.

### Step 2: DOM Injection Loop
1. Iterate through `DOMAIN_DATA`.
2. Map `domainId` to the global `window.MEMORY_MAP` to fetch real-time artifact counts.
3. Call `buildCardHTML()` per domain and append to `#domain-grid`.
4. **Mandatory Step**: Call `lucide.createIcons()` after the loop finishes.

### Step 3: Multi-Chart Rendering
- **Sparklines**: For each card, initialize a `Chart.js` bar or line chart.
- **Gradient Filling**: `AI/Human Step` -> Use `ctx.createLinearGradient(0, 0, 0, 40)` to fill line charts with neon cyan transparency.

### Step 4: Domain Selection Transition
1. User clicks a card -> `selectDomain(d)` triggered.
2. **Title Scaling**: `scaleConsoleTitle()` matches text width against a `280px` threshold, decrementing `fontSize` by `0.05rem` (min `0.5rem`) for perfect header fit.
3. Sidebar body is re-rendered via `buildSidebarHTML(d)`.
4. The "Traffic Chart" inside the sidebar is initialized via `renderSidebarChart(d)`.
5. **Step**: Previous chart instance MUST be destroyed via `activeConsoleChart.destroy()` before re-rendering.

### Step 5: Advanced Interactive Engines
1. **Sentinel Tooltips**: Event delegation with 300ms delay.
2. **Notification Center**: `initNotifications()` manages the dropdown and link to the central modal.
3. **Settings Engine**: `showSettingsMenu()` builds the config UI and wires sliders/toggles to `applySettingEffect()`.
4. **KPI Jitter Logic**: `initDataRefresh()` creates a live diagnostic feel.
   - **Math**: `jitter = (Math.random() - 0.5) * 4`.
   - **Persistence**: Base values are stored in `el.dataset.base` to prevent value drift over time.
   - **Frequency**: Controlled by `CONFIG.refreshRate` (default 2.5s).
5. **Sidebar 3D Flip**: Header icon click triggers perspective shift.

### Step 6: Modular Map Initialization (Deep Dive v2.3)
1. **Core Load**: `map_core.js` initializes the Three.js scene, camera, and renderer.
2. **Element Injection**: `map_elements.js` builds the 3D globe, clusters, and atmospheric layers.
3. **Operational Readiness**: `map_ops.js` registers the `Deep Scan` and `Lockdown` global functions.
4. **UI Binding**: `map_ui.js` connects HTML buttons to the internal JS logic.
5. **Orbital Activation**: `map_orbital.js` injects the 3D ring system and starts the label projection loop.
6. **Visual Sync**: `map_visual_sync.js` ensures `window.CONFIG` changes are immediately reflected in the 3D shaders.
