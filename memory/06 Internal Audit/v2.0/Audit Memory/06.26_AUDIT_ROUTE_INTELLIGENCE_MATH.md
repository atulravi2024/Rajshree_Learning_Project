## 🌎 GEODESIC CALCULATION ENGINE

The core of the navigation system relies on the **Haversine Formula** to compute true "Great Circle" distances on a 3D sphere.

### 1. The Haversine Core (`calculateGlobeDistance`)
- **Physics**: Earth's radius fixed at `6371 KM`.
- **Logic**: 
    - Converts `Lat/Lon` to `Radians`.
    - Computes `$\Delta Lat$` and `$\Delta Lon$`.
    - Uses `Math.atan2(Math.sqrt(a), Math.sqrt(1-a))` to derive the central angle ($c$).
    - Result is rounded to the nearest kilometer.

### 2. Path Alignment Scaling (`calculateActualDistance`)
- **Linear Adjustment**: The "actual" distance is modified based on the visual curvature and altitude arc:
    - **Straight**: 1.00x ratio (Physical baseline).
    - **Curve**: 1.15x multiplier (Adds 15% distance for arched altitude).
    - **Circle (Orbital)**: 1.30x multiplier (Adds 30% distance for full-globe tracks).

## 🤖 DETERMINISTIC INTEL GENERATOR (`generateLocationIntel`)

To fulfill the "Deep Dive" metadata requirement without external API dependencies, the system uses a character-code deterministic engine.

- **Mechanism**: Sums the ASCII values of the location name string to create a `seed`.
- **Variance**: Uses `Math.abs(Math.sin(seed * mult))` as a pseudo-RNG for consistent yet unique data points.
- **Dynamic Dataset Fields**:
    - **Area**: 200–1,700 KM². (`areaTypes` registry: URBAN, RURAL, INDUSTRIAL, METROPOLITAN, SEZ, TECH-HUB).
    - **AQI**: 15–195 range.
    - **Income**: $12k–$87k per capita.
