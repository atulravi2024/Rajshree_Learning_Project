## 🧱 MODAL ORCHESTRATION (`map_ui.js`)

The framework utilizes a centralized class-based state machine for UI overlays and modal interactions.

### 1. Component Registry
- **Modals**: `#modal-audit-logs`, `#modal-system-info`.
- **Target Class**: `.hud-modal` with `.open` to trigger CSS visibility.
- **Cleanup**: `clearInterval(window._infoModalInterval)` on close to prevent memory accumulation.
- **Minimize Logic**: Toggles `.ui-minimized` on `document.body` with a **520ms** timeout before dispatching the `resize` event to synchronize the 3D renderer.

### 2. Operational Durations
Exact durations used for UX "Audit Resonance":
- **Deep Scan**: 4,000ms execution window (Animated rotation: 0.018 rad/frame).
- **Mode Transitions**: 600ms to 1,200ms GSAP-like easing intervals.
- **Telemetry Refresh**: 1,500ms cycle for modal data updates.

## 📐 SIDEBAR COLLAPSE ORCHESTRATION (`map_ui_utilities.js`)

The Map sidebar system is designed for maximum screen-space utility, featuring a nested collapsibility logic (`initNavCollapsing`).

### 1. Element Delegation
- **Trigger**: Clicks on `.sidebar-toggle-label` or `.expand-btn`.
- **Target**: Closest `.holo-widget` or `.sidebar-section`.
- **Logic**: 
    - Toggles the `.collapsed` and `.hidden` classes via `initCardCollapsibility`.
    - Updates `window._mapTabState` to track active view context.

## 🛠️ UI UTILITY FRAMEWORK (`map_utils.js`)

A suite of helper functions for consistent DOM manipulation and data formatting in the holographic audit environment.

### 1. DOM Lookup Helpers
- **`byID(id)`**: Optimized alias for `document.getElementById`.
- **`qs(sel)`**: Optimized alias for `document.querySelector`.
- **`qsa(sel)`**: Optimized alias for `document.querySelectorAll`.

### 2. Data Formatting Logic
- **`formatNum(n)`**: Rounds and adds standard numeric comma-separators for audit metrics.
- **`formatCoord(lat, lon)`**: Formats raw lat/lon pairs into "XX.XX° N, XX.XX° E" strings for HUD display.
