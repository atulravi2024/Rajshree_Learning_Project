## рЯПЧпЄП SYSTEM ARCHITECTURE

The **Orbital Settings** replace the legacy static sidebar with a dynamic, immersive experience using 5 orbiting satellites. Each satellite represents a specific configuration domain and is linked to the shared project state (`window.CONFIG`).

### 1. The Satellites (Five Modules)
- **System Ops**: Core engine settings (Refresh Rate, Auto-Sync).
- **Visual Protocol**: Aesthetic overrides (Neural Glow, Scanlines, Glare).
- **User Gateway**: Auditor identity and activity logs.
- **Neural Audio**: HD audio script volume and interaction bleeps.
- **Security Guardrails**: Safety filters, data masking, and system lockdown.

### 2. Orbital Ring Mechanics
- **Ring 1**: Radius 118, Color `0x00f0ff` (Cyan), Speed `0.002`.
- **Ring 2**: Radius 130, Color `0xfacc15` (Amber), Speed `-0.0015`.
- **Ring 3**: Radius 143, Color `0x00f0ff` (Cyan), Speed `0.001`.
- **Blend Mode**: `THREE.AdditiveBlending` for holographic intensity.

## вЪЩпЄП LOGIC & SYNC (Step-by-Step)

### Step 1: Activation Logic
1. Triggered via the **Navbar Settings Gear icon**.
2. Globe scales to **0.65x** to make room for orbits.
3. `orbitalGroup.visible = true` and `.sat-label` elements are injected/revealed.

### Step 2: Interactive Panels
1. **Module Selection**: Clicking a satellite label triggers `openOrbitalPanel(id)`.
2. **Panel UI**: Glassmorphic holographic panels containing sliders (`input[type=range]`) and switches.

### Step 3: Global State Sync
1. **Change Detection**: Sliders use `input` listeners to call `applySettingEffect(id, val)`.
2. **Persistence**: Changes are synchronized across all modules (Dashboard, Map, Terminal) via the global `window.CONFIG` object.
3. **Immediate Feedback**: visual parameters (e.g., `--glow-opacity`) are updated in real-time on `document.documentElement`.
