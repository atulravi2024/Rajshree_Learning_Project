### 1. The `CONFIG` Persistence (`shared/settings.js`)
The system utilizes a 12-property global configuration object, synchronized with `localStorage` for cross-session continuity:
- **Registry Keys**: `fp_theme`, `fp_audit_depth`, `fp_glow`, `fp_scanline`, `fp_glare`, `fp_grid`, `fp_glass`, `fp_chroma`, `fp_refresh`, `fp_sync`, `fp_volume`, `fp_haptics`.
- **Initialization**: `loadSettings()` merges user-overrides with the `DEFAULT_CONFIG` baseline upon `DOMContentLoaded`.

### 2. Physical HUD Logic
- **Volume Visualization**: `#vol-indicator` manages 10 dynamic `<span>` elements. Height values are derived from a logarithmic curve to simulate professional hardware response.
- **Animation Reflow**: Employs the `void nameEl.offsetWidth` technique to bypass browser optimization and force-trigger the "Success Flash" after name changes.
- **Haptic Pulse**: `hapticPulse()` executes a 300ms global strobe across the primary HUD elements to confirm high-priority state changes.
