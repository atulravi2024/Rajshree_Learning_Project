## ⚙️ INTERACTIVE CONTROLS
The Settings Sidebar (Neural Console in `settings` mode) exposes these controls:

### 1. Visual Protocol (Sliders & Select)
- **Theme Select**: Swaps `body` classes (e.g., `theme-ghost`, `theme-synth`).
- **Chroma Distortion**: `.chroma-distort` class uses `--chroma-shift` to apply `hue-rotate` and `sepia` filters for high-stress HUD effects.
- **Range Sliders**: Update CSS variables (e.g., `--neural-glow-opacity`) in real-time using normalized floats (0.0-1.0).

### 2. Security & Neural Audio
- **Protocol Lock**: `.protocol-locked` class disables pointer events on the main portal via CSS.
- **Data Masking**: `.data-masked` applies a `-var(--data-blur-amount)` (6px) filter to all `.kpi-value` spans.
- **Volume Indicator**: `#vol-indicator` uses 10 dynamic `<span>` bars with a custom height curve array.
- **Haptic Pulse**: `hapticPulse()` triggers synchronized glare/nav flashes.

### 3. User Gateway (Auditor Profile)
- **Name Logic**: `void nameEl.offsetWidth` is used to force-restart the "Success Flash" animation on rename.
- **Activity Logs**: Integrated link to the Central Alerts Modal.

## 🤖 AI DIRECTIVES
- **Reflow Management**: Always use the "offsetWidth" trick when re-applying `name-updated` or `haptic-pulse` to ensure the animation re-triggers without a browser-cached state.
- **Dynamic Refresh**: `initDataRefresh()` uses an interval in `dashboard.js` to jitter KPI values.
