# Current Project Architecture

This is the exact markup representation of how your project is currently structured after our most recent cleanup. 

While the "root" folder is cleaner than it used to be (with floating Excel files pushed into folders), it still suffers from **fragmentation**. Backend tools, audio software, and web assets are all mixed side-by-side at the top level of the project.

## The Flaws in this Structure:
1. `Audio Demo` is floating in the root instead of being housed with the other backend/database scripts.
2. `project_docs` breaks GitHub's ability to read `README.md` natively since it's hidden in a folder.
3. The actual Frontend Web App (`css`, `js`, `assets`, `index.html`) has no dedicated container, making it hard to deploy to a server without accidentally deploying your entire backend database alongside it.
