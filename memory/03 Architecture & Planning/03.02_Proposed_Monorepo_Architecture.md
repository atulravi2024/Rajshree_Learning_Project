## Proposed Monorepo Architecture

This markup represents the **Absolute Best** structural organization for a modern, scalable learning application like Rajshree. 

This model introduces the `src` (source) pattern, which is the industry standard for separating code that runs on the web from the complicated backend infrastructure used to build it.

### Why is this the Ideal Structure?
1. **Pristine Root Directory:** Your root folder is strictly reserved for global project configuration and GitHub metadata, exactly as it should be.
2. **True Separation of Concerns:** Your live application code (`src`) sits entirely divorced from the complex systems used to build it. Deploying your website is as simple as uploading the `src` folder, with absolutely zero risk of accidentally exposing private backend databases.
3. **Unified Backend Tooling:** All database generators, Python Audio scripts, and Master Excel files are consolidated into one massive, organized `tools` directory. This makes long-term maintenance significantly easier.
4. **Restored GitHub Native Rendering:** `README.md` and `CONTRIBUTING.md` are restored to their rightful place in the root directory, fixing the GitHub native rendering issues.
