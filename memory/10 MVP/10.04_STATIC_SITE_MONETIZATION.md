# 💰 Serverless Monetization: GitHub Pages Guide
*Version: 1.0 | Status: Production-Ready | Prepared for: Rajshree Learning Project*

Implementing a paywall on **GitHub Pages** (Static Hosting) without moving to a paid server is 100% possible using the **"Serverless Triangle"** approach.

---

## 📐 The Architecture (Free Tier Stack)

Since GitHub Pages cannot run PHP, Node.js, or Python, we use third-party APIs to handle the "Brain" of the operation:

1.  **Firebase Auth (Free):** Handles User Sign-in (Google/Email).
2.  **Firebase Firestore (Free):** A database to store who has paid.
3.  **Razorpay (Free Signup):** Handles the actual money transaction.
4.  **GitHub Pages (Free):** Hosts your HTML/CSS/JS.

## 🏗️ Step-by-Step Implementation

### Step 1: Set up a BaaS (Backend-as-a-Service)
While Firebase is the most popular, other providers offer specialized "Always Free" tiers that might suit your needs better.

### Step 2: Integrate Razorpay Payment Button
Razorpay provides a "No-Code" payment button that works perfectly on static sites.
1.  Go to **Razorpay Dashboard** -> **Payment Buttons**.
2.  Create a button for "₹99 Monthly Subscription".
3.  Copy the `<form>` code they give you.
4.  Place this code inside your `PaywallOverlay` (defined in Priority 2 docs).

### Step 3: Secure the Content (Client-Side Locking)
Since we are on a static site, we "hide" the data using JavaScript. While a pro developer could bypass this, for 99% of your users, it is perfectly secure.

#### Sample Logic (`router.js`):
```javascript
// Check if user has premium flag in your chosen BaaS
if (user.premium === true) {
    window.userHasPremium = true;
    unlockCategories();
}
```

## 📊 BaaS Alternatives & Comparison Table (Always Free Tiers)

| Provider | Auth Limit | Database Limit | Bandwidth/Month | Cloud Functions | Best For |
|:---|:---|:---|:---|:---|:---|
| **Firebase** | Unlimited (Core) | 1GB Storage / 50k Reads | 10GB | No (Paid only) | Unified Ecosystem |
| **Supabase** | 50,000 MAU | 500MB Postgre | 5GB | Included (Edge) | SQL & Complex Queries |
| **Appwrite** | 75,000 MAU | 2GB Total | 10GB | Included | Best All-in-One |
| **Clerk** | 10,000 MAU | User Metadata only | N/A | N/A | Premium Auth UI |
| **Pocketbase** | Unlimited (Self) | 2GB (on free host) | Depends on Host | JS Hooks | Power Users |

### 🔍 Deep Dive: Always Free vs. Free Tier

- **Firebase (Google):** Its "Spark Plan" is **Always Free**. You get plenty of reads/writes for a kids' app. Great if you already use Google services.
- **Supabase (Open Source):** The best "Free Tier" for developers. It gives you a real PostgreSQL database for free forever (for 2 projects).
- **Appwrite Cloud:** Extremely generous. Its free tier is designed to help small startups launch without costs.
- **Clerk:** If you don't need a database for anything other than "Has this user paid?", Clerk's free tier is the most polished for handling logins.
