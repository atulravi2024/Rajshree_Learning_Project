## 🌿 Branch Strategy

| Branch | Purpose |
|:---|:---|
| `main` | Stable, production-ready code. Primary release branch for **Rajshree Learning**. |
| `dev` | Active development branch for both learning core and frontier infrastructure. |
| `feature/*` | Short-lived branches for new Hindi categories or protocol upgrades. |
| `fix/*` | Hotfix branches for urgent bug fixes. |

### Rules:
- **Never commit directly to `main`.**
- Always open a **Pull Request (PR)** from `dev` → `main` with a descriptive title.
- Delete feature/fix branches after merging.

## 📝 Commit Message Convention

Use the following format for all commits:

```
<type>(<scope>): <short description>
```

### Types:
| Type | When to Use |
|:---|:---|
| `feat` | Adding a new feature, category, or protocol module |
| `fix` | Bug fixes (Logic or UI) |
| `style` | CSS/UI changes (No logic change) |
| `refactor` | Code restructuring/Optimization |
| `docs` | Updating memory files, README, or audit logs |
| `audio` | Adding/Updating audio assets via pipeline |
| `chore` | Config, tooling, or deployment changes |
