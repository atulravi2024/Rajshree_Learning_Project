## 🌿 Branch Strategy

| Branch | Purpose |
|:---|:---|
| `main` | Stable, production-ready code. Only merge from `dev` after full testing. |
| `dev` | Active development branch. All feature work happens here. |
| `feature/*` | Short-lived branches for specific new features (e.g., `feature/new-category`). |
| `fix/*` | Hotfix branches for urgent bug fixes (e.g., `fix/audio-sync`). |

### Rules:
- **Never commit directly to `main`.**
- Always open a **Pull Request (PR)** from `dev` → `main` with a descriptive title.
- Delete feature/fix branches after merging.

## 📝 Commit Message Convention

Use the following format for all commits:

```
<type>(<scope>): <short description>
```

### Types:
| Type | When to Use |
|:---|:---|
| `feat` | Adding a new feature or category |
| `fix` | Bug fixes |
| `style` | CSS/UI changes only (no logic change) |
| `refactor` | Code restructuring (no feature/fix) |
| `docs` | Updating memory files, README, or comments |
| `audio` | Adding or updating audio assets |
| `chore` | Config, tooling, or dependency changes |

### Examples:
```
feat(sankhya): add Tables M3 sub-category
fix(slideshow): resolve double audio trigger on page switch
style(navbar): increase logo size to 95px
docs(memory): update GITHUB_MEMORY with branch strategy
audio(varnamala): add 12 missing Vyanjan audio files
```

## 🚀 Deployment

| Item | Detail |
|:---|:---|
| **Hosting** | GitHub Pages (via GitHub Actions) |
| **Deploy Branch** | `main` |
| **Workflow File** | `.github/workflows/deploy-pages.yml` |
| **Deploy Target** | `frontend/` directory is automatically packaged by the Action |
| **Entry Point** | The payload is restructured so `frontend/public/index.html` becomes root |
| **Assets Path** | Restructured to root: `assets/audio/` (~369 MP3 files) |
| **CSS/JS Logic** | Restructured to root: `src/css/` and `src/js/` |

> ⚠️ **Pre-Deployment Checklist for GitHub Actions:**
> - Ensure all asset paths in your code use relative links (e.g., `../src/css/main.css` or `assets/audio/`).
> - The deployment script *automatically* rewrites `../src/` to `src/` in `index.html` during build. Do NOT change this in your source code.
> - Enable **GitHub Pages** in repo Settings -> Pages -> Source: select **GitHub Actions**.
> - Do not push `build_payload` (it is generated ephemeral on GitHub's servers).
> - ✅ **No Git LFS needed** — audio files are ~32 KB each.

## 🔄 Sync Workflow (Daily)

```
1. git pull origin main              ← Sync latest from remote
2. Make changes locally
3. git add .
4. git commit -m "type(scope): description"
5. git push origin main              ← Push to main (or dev if branching)
6. (When using branches) Open PR: dev → main
```

### First-Time Push (if not yet pushed):
```
git remote add origin https://github.com/atulravi2024/Rajshree_Learning_Project.git
git branch -M main
git push -u origin main
```
> ✅ Remote `origin` is already configured in `.git/config`. No need to `remote add` again.
