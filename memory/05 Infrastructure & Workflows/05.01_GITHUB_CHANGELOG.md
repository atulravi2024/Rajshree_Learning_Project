## Architecture Decisions

### Why a Separate GitHub Memory Folder? (v1.0)
- The existing `memory/01 Current State/` files (`PROJECT_MEMORY.md`, `PROJECT_CSS.md`, etc.) focus on **app logic and architecture**.
- GitHub/repo concerns (branching, commits, deployment) are **separate from app concerns** and should be tracked independently.
- A `memory/05 Infrastructure & Workflows/` subfolder keeps repository management knowledge isolated and easy to load on-demand.

## Useful Git Commands Reference

```bash
# Check current status
git status

# See commit history (short)
git log --oneline -10

# See what remote is configured
git remote -v

# Push current branch to GitHub
git push origin main

# Pull latest from GitHub
git pull origin main

# Check which files would be ignored by .gitignore
git check-ignore -v *

# See total repo size
git count-objects -vH
```
